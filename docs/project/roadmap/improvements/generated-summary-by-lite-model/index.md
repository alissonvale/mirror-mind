# Generated summary by lite model

**Status:** ✅ Shipped 2026-04-19
**Source:** [Identity Lab spike §9.2](../../spikes/spike-2026-04-18-identity-lab.md#92-generated-summary-by-lite-model-for-cards-and-routing)

## Goal

Unify the two short-form representations of an identity layer (Cognitive Map cards + reception descriptor) via a single summary generated by a lite model, persisted in the DB and reused everywhere.

Before this change, two separate mechanisms each had a limitation: `firstLine` (in `map.tsx`) surfaced markdown headers like `# Alma` on the card; `extractPersonaDescriptor` (in `personas.ts`) used the first non-header line, which was ambiguous between Template B personas (e.g. `tecnica` and `dba` both starting with the same independence declaration).

## Status

- [x] Schema: new `summary TEXT` column on `identity` (with ALTER TABLE migration in `migrate()`)
- [x] DB helper: `setIdentitySummary(db, userId, layer, key, summary)`
- [x] Generator: `server/summary.ts` — `generateLayerSummary()` calling the existing `title` model role
- [x] Trigger on Save: `handleWorkshopSave` and persona create/update handlers fire-and-forget summary generation
- [x] Regenerate endpoint: `POST /map/:layer/:key/regenerate-summary` and admin variant `POST /map/:name/:layer/:key/regenerate-summary`
- [x] Workshop UI: summary block + "Regenerate summary" button below the header
- [x] Cognitive Map cards: `StructuralCard` accepts `summary` prop, prefers it over `firstLine`
- [x] Reception descriptor: `extractPersonaDescriptor` accepts `IdentityLayer`, prefers `summary` over first line
- [x] Tests: 11 new tests (db, personas)
- [x] Worklog entry
- [x] Radar entry removed
