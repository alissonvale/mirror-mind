# v0.7.0 — Tuning the Voice

*19 April 2026*

**TL;DR**
- `ego` splits into three keys — **identity** (who I am), **expression** (how I speak), **behavior** (how I act) — so problems of form, method, and positioning can be diagnosed separately.
- Composition order reorganized around two clusters: identity first (soul → identity → persona), then form (behavior → expression). Expression sits last so its absolute rules keep recency weight over any persona content.
- Every identity layer carries an auto-generated **summary** — used on Cognitive Map cards, as the descriptor the reception router sees, and as a hover tooltip on persona badges. A dedicated prompt leads persona summaries with domain signal (for routing) followed by posture (for display).
- **Composed prompt drawer** on the Cognitive Map and on every layer workshop. Choose persona + adapter, see exactly what the LLM receives.
- Reception gets sharper: meta questions ("Quem é você?") route to the base voice; action verbs ("write", "draft") dominate conceptual topics; the prompt is persona-agnostic (no hardcoded keys).
- New `evals/` top-level folder — end-to-end LLM quality probes, distinct from `tests/`. First eval: routing, 22 probes, threshold 85%.
- Chat polish: calm typing indicator replaces the white waiting rectangle, sidebar collapses on desktop, Cognitive Map gets subtler card previews with `read more →`, favicon 404 silenced, lab mode gated behind `?lab=1`.

## Where we were

v0.6.0 opened the Admin Workspace — the admin gained a real surface to see and operate this mirror from the browser. But when Alisson sat down to iterate on his own identity in the days that followed, the friction was elsewhere. The mirror's voice was close, not quite his. Refining it meant editing identity layers blind: write a rule into `ego/behavior`, send a message through the chat, read the response, guess what shifted, repeat. No feedback loop. No way to see what the LLM was actually reading. Problems of conduct and problems of form bled into each other inside the same layer. Persona routing was a black box.

This release closes those gaps. Most of the work lands **Identity Lab spike** findings (18 April) and the voice-probing session that followed. The mirror's structure becomes more legible, more iterable, more in tune.

## What we did

**The ego splits into three keys.** `ego/behavior` used to carry two concerns stacked in the same text: *how I act* (conduct, posture, antagonism, method) and *how I speak* (format, vocabulary, punctuation rules, anti-listicle). The spike exposed that every time a response failed, it was hard to say whether the problem lived in method or in form. The split makes the distinction structural: `ego/identity` (operational positioning, who I am in the day-to-day), `ego/expression` (how I speak, the absolute rules about em-dashes and listicle, the vocabulary palette), `ego/behavior` (conduct, posture, method, the "ressoar antes de responder" kind of rule). Each layer has its own workshop surface, its own summary, its own place in the composed prompt.

**Composition reorganized into identity and form clusters.** The old order — self → ego → persona — put persona last. Under recency bias, persona content dominated the expression rules that preceded it. After voice-probing revealed the listicle pattern leaking through even on a stronger model, the composition was regrouped. Now: the **identity cluster** comes first (self/soul → ego/identity → persona), treating persona as a specialization of identity rather than an afterthought. Then the **form cluster** (ego/behavior → ego/expression), invariant across personas, with expression in the final position for maximum recency over its absolute rules. Display order in the Cognitive Map (identity → expression → behavior) is a different ordering, tuned for human reading.

**Summaries on every layer.** Each identity layer gained a new `summary TEXT` column, populated by a cheap lite model role (the same `title` model already used to label sessions). The summary powers three consumer surfaces: the preview text on Cognitive Map cards (replacing the previous "first line of content" heuristic that surfaced markdown headers like `# Alma`), the descriptor reception uses to route between personas (replacing first-line extraction that made Template B personas indistinguishable), and the hover tooltip on persona badges. Summaries regenerate fire-and-forget on Save. A "Regenerate all summaries" bulk action lives on the Personas card.

**Persona summaries are routing-aware.** The first iteration of the summary prompt produced formulaic openings — every summary began "Esta camada opera" and ended "Distingue-se por...". The rewrite bans those patterns explicitly, adds good/bad few-shot pairs, and branches on `layer === "persona"`: for personas, the first clause must name **domain and activation triggers** concretely ("Personal finance: spending, runway, balances...") so the router has a clear signal, followed by posture ("calm reading of numbers as data, not verdict"). For self/ego, the prompt optimizes for essence-distillation only. Language-matching rule at the end in bold: Portuguese content produces Portuguese summaries, English produces English. The prompt itself stays in English (D7).

**The composed-prompt drawer.** A slide-in drawer anchored to the right edge of the Cognitive Map shows the full system prompt for a chosen `(persona, adapter)` pair. Two dropdowns — persona (none + each persona key rendered from the DB) and adapter (none + web + telegram + cli) — update the view in place. The old inline preview pane inside each layer workshop is gone; the workshop is now single-column, focused on the textarea. The same drawer lives on every workshop page too, opened from the breadcrumb. "What does the LLM actually see right now?" is a question you can now ask from anywhere in the identity surface, one click away.

**Reception is clearer and more universal.** Three concurrent improvements. First, the prompt now lists explicit null-cases with examples — greetings, meta-questions about the mirror itself, open existential questions without a domain — so "Quem é você?" stops mis-routing to a best-guess persona and defers to the base ego voice. Second, an **action-verbs-dominate-topic** rule: when the user asks for production of an artifact (write, draft, compose), the verb defines the work and the topic is just the subject matter — so "write a text on antifragility" routes to the writing persona even though the topic is conceptual. Third, the prompt is now **persona-agnostic** — the old version had Alisson's persona keys (`escritora`, `tesoureira`, `pensadora`) hardcoded into the routing examples; the new version describes categories abstractly and instructs the model to select from the runtime-rendered persona list. Other users with different persona sets get the same routing logic for free.

**Evals become first-class infrastructure.** LLM behavior isn't unit-testable. A broken reception prompt wouldn't fail any test — but it would silently route messages to the wrong persona for weeks. Mirror Mind now has a top-level `evals/` folder, sibling to `tests/`, with its own runner (`evals/_lib/runner.ts`), its own types (`evals/_lib/types.ts`), and its own documentation (`docs/process/evals.md`). Each eval is a standalone `tsx` script, hits real LLMs and real DBs, prints a report, and sets an exit code based on a configurable threshold. The first eval — `routing.ts` — validates reception with 22 probes covering clear-domain routing, previously ambiguous cases, meta/null handling, and the action-verb rule. Run with `npm run eval:routing`. Current score: 20/22 (91%) against threshold 85%. The two misses are genuinely ambiguous cases where the "correct" persona is subjective.

**Voice-probing surfaced a stubborn LLM limit — captured, not fought further.** Three layers of reinforcement (composition reordering, model swap to Claude Haiku 4.5, an experimental `FINAL_REMINDER` block) could not prevent models from producing disguised-listicle responses when the question had plural enumeration grammar ("quais são as coisas mais importantes..."). The em-dash absolute rule holds; the listicle rule does not hold under that question shape. Documented in the compose-order improvement plan as an accepted limit — prompt engineering cannot fully override strong priors for structured responses; the mitigation path is fine-tuning or a stronger model, not more prompt rules. The `FINAL_REMINDER` block was removed; the composition reordering stayed.

**Chat and map polish.** The streaming assistant bubble used to render as an empty white rectangle while waiting for the first SSE delta. Now it starts with three slow pulsing dots in soft terracotta (1.4s animation, 0.2s stagger) that the first delta replaces naturally — calm and appropriate, not the fast bouncing dots of chat apps. A sidebar toggle button lets the user collapse the sidebar on desktop to see the map wider, with smooth transitions. Card previews on the Cognitive Map got lighter typography (`0.76rem`, weight 300, soft terracotta) and a three-line truncation with a `read more →` that only appears when the preview actually overflows. The sidebar brand ("Mirror Mind") is now a link to `/mirror`. The favicon 404 is suppressed via an inline data URI. The Lab mode checkbox on `/mirror` is gated behind `?lab=1` — the normal UX stays clean.

**Source-of-truth file made the split practical.** The Identity Lab POC kept conduct and expression as two sections inside the same `ego/behavior` as an interim measure. With this release, Alisson's source-of-truth file at `identity-lab/identity.md` was split manually into `ego/behavior` and `ego/expression` blocks and synced via `identity-lab/sync.mjs`. That script — simple parser that reads between `═══` delimiters and UPDATEs the identity table — is gitignored but real. New users get `ego/expression` seeded from template on creation.

## Upgrading

### From v0.6.0

```bash
git pull && npm install && systemctl restart mirror-server
```

On first boot after upgrade, migrations run automatically:
- `identity.summary` column is added via `ALTER TABLE`. Existing rows have `summary = NULL`; consumers fall back to previous behavior (first line of content) until the user saves or hits Regenerate.

After upgrade, recommended (not automatic):

**Regenerate all summaries.** On the Cognitive Map's Personas card, click **regenerate all summaries**. This triggers `POST /map/personas/regenerate-summaries`, which runs the lite model across every persona in parallel. Do the same per-layer via the workshop's Regenerate button for soul, ego/identity, ego/behavior, ego/expression.

**Add `ego/expression` to your existing identity.** The new layer is seeded from template for new users. Existing users (only Alisson, in practice) start with no `ego/expression` row. The workshop at `/map/ego/expression` has the empty-state invitation. If you had expression rules mixed inside `ego/behavior`, move them to the new layer manually — there is no auto-migration, by design (the boundary between conduct and expression in existing content is a convention, not a guarantee, and an automatic split would mangle content that doesn't follow it).

**Consider swapping the `main` model to Claude Haiku 4.5.** Voice-probing showed it respects absolute rules (no em-dash) more consistently than DeepSeek Chat v3. Edit at `/admin/models` — `main` role, provider `openrouter`, model `anthropic/claude-haiku-4.5`. Not automatic: model choices are a matter of taste and cost profile, and the existing DeepSeek configuration continues to work.

### From older versions

Follow the v0.6.0 upgrade notes first (`models` table seed, `sessions.title` column). This release stacks cleanly on top.
