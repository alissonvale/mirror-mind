# v0.9.0 — Subscription, reconsidered

*21 April 2026*

**TL;DR**
- **[CV0.E3.S8 — OAuth credentials for subscription-backed providers](../project/roadmap/cv0-foundation/cv0-e3-admin-workspace/cv0-e3-s8-oauth-subscriptions/)** shipped. Five pi-ai OAuth providers wired (Anthropic, OpenAI Codex, GitHub Copilot, Google Cloud Code Assist, Antigravity) behind a paste-credentials UI at `/admin/oauth`. `models.auth_type` column + `resolveApiKey()` as the single seam every LLM call now goes through.
- **[CV0.E3.S6 — Budget as simulated subscription](../project/roadmap/cv0-foundation/cv0-e3-admin-workspace/cv0-e3-s6-budget-dashboard/)** shipped immediately after. Every LLM call logs real cost from OpenRouter's `/generation/{id}` endpoint. `/admin/budget` surfaces credit remaining, burn rate, 7-day projection, breakdowns by role + environment + model. Soft low-balance banner at 20%. Exchange rate is global, BRL display is per-admin.
- **Cost visibility is admin-only.** Regular users no longer see BRL cost on the Context Rail. Admins see USD or BRL per their own preference.
- **311 tests passing** (was 237). Zero regressions.
- **Migration is additive.** New tables (`oauth_credentials`, `usage_log`, `settings`), two new columns (`models.auth_type`, `users.show_brl_conversion`), one seed row. No data loss, no SQL required.

## Where we were

v0.8.1 closed with a question and a queued story. The question: *could we replace per-token OpenRouter billing with a subscription?* The queued story: CV0.E3.S8, the OAuth infrastructure to make that pivot possible. The most attractive path identified in the [spike](../project/roadmap/spikes/spike-2026-04-21-subscription-oauth.md) was Google Cloud Code Assist for Individuals — free tier, personal developer use, Gemini models at zero marginal cost.

The plan was clean on paper. The implementation was clean in code. The *reality* closed two of the three paths the spike relied on within hours of S8 landing.

## What we did

### S8 — the OAuth infrastructure, as spec'd

Five phases, committed individually, each green:

1. **Schema** — `oauth_credentials(provider PK, credentials JSON, updated_at)`, `models.auth_type` column via additive migration, full CRUD helpers.
2. **`resolveApiKey(db, role)`** wrapper — single seam replacing five call sites' direct `process.env.OPENROUTER_API_KEY` reads. For `auth_type='oauth'`, calls pi-ai's `getOAuthApiKey()`, persists refreshed credentials back, wraps failures as `OAuthResolutionError`. Reception's existing silent-fallback catches it; the Agent-based main paths wrap the wrapper in `getApiKey` callbacks that honor pi-agent-core's no-throw contract.
3. **`/admin/oauth`** — paste-JSON UI per provider. Surfaces configured/not, relative expiry, extra fields on the blob (e.g. `projectId`, `email`). Validation rejects malformed JSON and missing `refresh`/`access`/`expires` with clear flashes. Accepts both pi-ai's envelope shape (`{provider-id: {...}}`) and the inner object.
4. **`/admin/models` auth-type aware** — env/OAuth badge per role card; shared datalist of provider ids; inline warning with a link to `/admin/oauth` when an OAuth-configured role lacks credentials. `auth_type` derives implicitly from the chosen provider.
5. **Docs + review pass.**

Then acceptance testing. And the hypothesis started breaking.

### The hypothesis fell apart in hours

**Google Code Assist for Individuals — the primary target.** Logged in via pi-ai's `login google-gemini-cli`, pasted `auth.json` at `/admin/oauth`, pointed reception at `google-gemini-cli` / `gemini-2.5-flash`. Ran `npm run eval:scope-routing`. Results at 5-second timeout: **0/11 completing** (all hitting the timeout). Bumped timeout to 20s: **2/11 completing**, 9 still timing out. Bumped to 60s: the latency curve kept suggesting rate-limit or cold-start behavior the free tier couldn't swallow at the 11-probe burst the eval generates. Reception needs sub-5-second p95 to be useful — Google Code Assist delivered ~6s on the *fast* calls and silent failures on the rest.

Reverted reception to OpenRouter before the eval became validation theater.

**GitHub Copilot — the attractive fallback.** $10/month flat, rich model catalog via pi-ai (Claude Sonnet 4.6 with 1M context, GPT-5.4, Gemini 3 Pro). Excellent-looking backup. Then the user found [the GitHub blog post](https://github.blog/news-insights/company-news/changes-to-github-copilot-individual-plans/) announcing changes to individual Copilot plans that closed the pattern the spike relied on.

**Anthropic Claude Pro/Max via OAuth.** Still technically works but Anthropic has been tightening Claude Code OAuth's external-app surface for weeks. Using it today is a bet on a window that's observably closing.

What was left? Pay-per-token on OpenRouter, same as before S8.

### The pivot — S6 was written, in full, as a UX bet instead of an infrastructure bet

If the *infrastructure* path to flat-rate was blocked, the *experience* of flat-rate could still be delivered. OpenRouter supports prepaid credit. A dedicated account isolates mirror spending from any other OpenRouter app. The mirror already has full access to every LLM call — with one column added to a new table, it knows exactly what each call cost. With one page, it surfaces balance, burn rate, projection, and a low-balance banner. The result feels like a subscription from the admin's side, even though the billing substrate is unchanged.

Seven phases, committed individually, each green:

1. **`usage_log`** (one row per LLM call — tokens from pi-ai, cost from OpenRouter resolved async), **`settings`** (generic key/value, seeds USD→BRL rate), **`users.show_brl_conversion`** column.
2. **`server/openrouter-billing.ts`** — `getKeyInfo()` cached 60s, `getGeneration(id)` with exponential retry on 404 (OpenRouter's endpoint has a few-second lag after a call completes). Both degrade gracefully on transport failure.
3. **`logUsage()`** wired into all five call sites (reception, title, summary×2, main via web/telegram/api). Synchronous insert with NULL cost, background async reconciler patches the row with real cost via `/generation/{id}`. pi-ai exposes `AssistantMessage.responseId` — the OpenRouter generation id lives there, no fallback to `/activity` reconciliation needed.
4. **`/admin/budget`** — credit hero with progress bar (against the OpenRouter spending cap), this-month total, 7-day burn rate + projected days, breakdowns by role / environment / model. Preferences section with the rate editor and the BRL toggle.
5. **`MIRROR_ENV`** env var for dev/prod isolation, `X-Title: mirror-mind` + `HTTP-Referer` headers on every OpenRouter call, **soft low-balance banner** injected client-side on `/admin/*` when credit drops under 20%.
6. **Context Rail becomes admin-only for cost.** Regular users lose the BRL cost line entirely — it stays in DOM (data-hidden=true) to avoid layout shifts but renders nothing. Admins see USD or BRL per their `show_brl_conversion` preference.
7. **Docs, refactoring, status update.**

The cost of the whole scope-routing eval came in at **$0.07 USD** on Gemini 2.5 Flash. Extrapolated to typical single-user volume (~10 messages/day), a $10 prepaid deposit covers 2-3 months. That's the flat-rate *feel* — small enough to forget, visible when it matters, honest about what it actually is.

### Observed OpenRouter dashboard behavior

During S6 validation, we noticed the OpenRouter activity dashboard displays **HTTP-Referer** (the origin URL) in its "App" column, not `X-Title`. Both headers reach OpenRouter — pi-ai's openai-completions provider forwards custom headers intact — but the dashboard prefers the URL for visual grouping. Practical consequence: `MIRROR_BASE_URL=https://mirror.your-prod-domain` is what distinguishes installs in the OpenRouter dashboard, not the X-Title tag. The internal `usage_log` carries the full detail (role, env, model); OpenRouter's view is a secondary reference.

## Upgrading

### From v0.8.1

```bash
git pull
npm install
systemctl restart mirror-server
```

Then:

1. Add to `.env`: `MIRROR_ENV=prod` (or `dev` locally), `MIRROR_BASE_URL=https://your-origin` (or leave default).
2. At `openrouter.ai/settings/keys`, set a monthly spending cap on the mirror's key. Without a cap, `/admin/budget`'s hero and alerts can't calibrate.
3. Visit `/admin/budget` to verify the dashboard loads. Adjust the USD→BRL rate from the Preferences section. Toggle BRL display off if you track in USD.

Schema changes run automatically on boot via `migrate()`. No data loss, no SQL required.
