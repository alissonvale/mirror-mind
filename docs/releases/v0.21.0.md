# v0.21.0 — Two new dials, one cast that holds Alma

*1 May 2026*

**TL;DR**

- **[CV1.E10.S2 — Response length axis](../project/decisions.md#2026-05-01--response-length-is-a-session-level-axis-behind-an-advanced-disclosure-cv1e10s2)**. New `sessions.response_length TEXT NULL` column with values `brief / standard / full / null(=auto)`. The expression pass appends a length guide to the system prompt with explicit "length is the primary size constraint" framing so combinations like `essayistic + brief` actually compose a bounded reflection. Mode pill collapses into a single `Avançado / Advanced` disclosure that carries both mode and length; the summary shows `auto` when both axes are auto, `<mode>/<length>` otherwise. Header real-estate stays focused on what's identity-bearing (Cast + Scope); admin-feel controls live one tap behind a disclosure.
- **[CV1.E9.S6 — Soul Voice in the cast](../project/decisions.md#2026-05-01--soul-voice-is-a-cast-level-alternative-to-personas-cv1e9s6)**. New `sessions.voice TEXT NULL` column. When `voice = "alma"`, the cast renders a single ♔ avatar (warm-amber, `#b8956a`) instead of persona avatars and every turn forces the Alma composer regardless of reception's `is_self_moment` verdict. Mutual exclusion enforced in the same transaction as the write — setting voice clears `session_personas`; adding a persona via `/conversation/tag` clears voice. The cast `+` picker carries Soul Voice as a first-class entry at the top of the panel, distinct from the persona select. Per-turn `Send to…` overrides still work in both directions (persona on Alma cast, Alma on persona cast).
- **[Soul Voice label localizes properly](../project/decisions.md#2026-05-01--user-facing-soul-voice-label-is-locale-aware-cv1e9s6-follow-up)**. The 2026-04-27 vocabulary asymmetry decision had pinned "Voz da Alma" across all locales as a brand exception. Real use surfaced the cost — docs read incoherently with a Portuguese label embedded in English prose, en-locale users got a phrase they had no context for. Returns the term to the i18n table: "Soul Voice" in en, "Voz da Alma" in pt-BR. Internal identifiers (`is_self_moment`, `composeAlmaPrompt`, `voz-da-alma.ts` filename, story folder names) keep their Portuguese-derived shape — those are EN-internal codepoints, not user-facing labels.
- **Mobile chat overhaul.** Bubble padding reduced and the 80%-width cap dropped on assistant turns so long replies use available width; the assistant `.msg-body` gets `width: 100%` so its right edge aligns with the user bubble's. Mode panel anchors to the viewport with `position: fixed; top: 4rem; right: 0.5rem` instead of the small pouch element, fixing the off-screen overflow when the pouch wrapped to a left position. The segmented control stacks vertically on small viewports for bigger tap targets. Long bubbles collapse to 5 lines via `-webkit-line-clamp` with a Read-more affordance injected client-side; freshly-streamed bubbles start expanded, history-hydrated bubbles start collapsed. The rail (`Olhar dentro`) becomes a `position: fixed` bottom-sheet drawer on mobile so toggling visibly surfaces it instead of stacking it below the conversation thread off-screen.
- **All header forms async.** Mode, length, voice, tag, untag — every header form intercepted at the `submit` event and posted via `fetch`, replacing the default POST → 302 → full-page reload that bounced the user to the bottom of the chat with a cleared input. Mode/length: optimistic active-class toggle + summary recompute, panel stays open. Tag/untag/voice: post + DOMParser-based swap of the `.conversation-header` node, pool data attrs synced. Rail toggle moved to event delegation so it survives the header replacement. Native `form.submit()` fallback on fetch failure preserves the no-JS path.
- **Cast/scope pickers auto-submit on selection.** Picking a persona, org, or travessia from the inline `<select>` posts immediately on `change` — the explicit "Add" button is hidden when JS is active (`body.js-on` + CSS rule). One tap per addition instead of two. SSR keeps the button so the no-JS fallback still works.
- **Chat input persists to localStorage.** Every keystroke saves the draft under a single key; restored on page load when the input is otherwise empty; cleared on successful send. Defends against any path that triggers a reload (a stray non-async form, F5, accidental back-button, browser crash) — the in-progress message survives. The previous data-loss risk surfaced when the user was typing a long message and changed a header axis: the page reloaded and the input cleared.
- **Cast / Elenco header label returns.** The persona zone reverts from "Personas" back to its original "Cast" (en) / "Elenco" (pt-BR) now that the zone holds two distinct voice options (persona pool OR Soul Voice). "Personas" became misleading the moment Alma joined the zone.
- **Bug fixes from real use.**
  - **Scope auto-seed gate is symmetric across axes** (org / journey now seed when their pool is empty regardless of turn number; previously gated to first-turn-only, which left turn-2-and-beyond classifications stranded on entry meta only — prod 28 Apr 2026 hit this).
  - **Bubble badge follows transition rule, not pool membership** (the `⌂ org` and `↝ journey` badges show on the turn whose scope differs from the previous assistant turn's; symmetric with persona's `newPersonasThisTurn`. The previous pool-suppression rule masked the seed turn's marker once junctions were correctly populated).
  - **Per-turn override beats session voice** (cast=Alma + `Send to mentora` now correctly routes that one turn through mentora instead of Alma — earlier the session voice clause fired even when a persona was forced).
  - **Auth Secure cookie is dynamic** (mobile-via-LAN-IP login was silently dropping the response cookie because browsers accept `Secure` on `localhost` HTTP as a special case but not on `192.168.x.y` HTTP. Detection now reads `X-Forwarded-Proto` or the request URL's protocol — secure: true on HTTPS, false on HTTP. Works in localhost dev, LAN-IP dev, and proxied prod with zero config).
  - **Mobile scope `+ Add` no longer spills off-screen** — each scope group occupies its own row on narrow viewports.
- **916/916 tests passing**, up from 729 at v0.17.0. New unit suites: `tests/scope-seed.test.ts`, `tests/scope-transition.test.ts`, `tests/response-length.test.ts`, `tests/session-voice.test.ts`. Existing suites expanded: web routes, expression, reception. The `tests/voz-da-alma-calibration.test.ts` describe block + reception assertion translated to "Soul Voice" alongside the i18n correction.

## What's parked for next session

**CV1.E11 — Cenas (Scenes) as primary entity.** A feature design conversation at the tail of this cycle surfaced a pivot worth its own session. The original sketch (cena = savable bundle of header configurations, with picker on "Nova" and inline injection of briefing into prompt) treated cena as a shortcut over the existing model — personas, orgs, travessias still came first; cena bundled them.

Walking a real user case (`Aula Nova Acrópole` — weekly philosophy class, briefing-rich, no perfect persona in inventory) revealed the inversion: **the scene IS the model**, not a feature on top of it. Personas, organizations, journeys are consequences of scenes — they emerge in service of specific recurring conversations the user wants to have. The product's first-run UX changes accordingly: new tenants describe a recurring conversation, the system pulls the scene out and creates supporting entities inline. The user voiced wanting to archive his existing personas and recreate them through scenes.

A four-story arc was sketched (cena CRUD with inline creation + picker, save-current-as-cena, cena-first onboarding, archive existing personas) but order and form-shape are undecided. Full notes in [`docs/design/scenes-pivot.md`](../design/scenes-pivot.md); a brainstorm session opens next time.

## Where we were

v0.20.0 (*Three weights, one minimal compose path*) closed the trivial turn class — courtesies and protocol exchanges that don't earn the always-on behavior baseline now route to a minimal composer that returns only the adapter instruction. The pipeline grew its third weight class: heavy (Alma), medium (canonical), light (trivial). CV1.E10 (Token Economy) opened with that story.

What stayed unchanged at v0.20.0: the session had **one** dial (mode) carrying response shape, with persona vs Alma decided per-turn by reception. Mobile chat had visible rough edges (narrow bubbles, mode panel that overflowed off-screen). Auth-on-LAN-IP didn't work because of the Secure cookie default. And every header form reload was a small data-loss risk for in-progress input.

v0.21.0 closes those on multiple fronts.

## What we did

### CV1.E10.S2 — Response length axis + Advanced disclosure

The `essayistic` mode produced fuller responses than the user wanted in some moments. Mode and length are orthogonal axes — the user could imagine `essayistic + brief` (bounded reflection, dense in a small space) and `conversational + full` (long plain paragraph), but the prior pipeline only had mode. Adding length without bloating the always-visible header meant grouping both into a single collapsed disclosure.

`sessions.response_length TEXT NULL` mirrors the `response_mode` shape. New helpers `getSessionResponseLength` / `setSessionResponseLength`, ownership semantics identical. New route `POST /conversation/response-length` mirror of `/conversation/response-mode`.

The expression pass picks up an optional `length` parameter on `ExpressionInput`. When non-null, `buildSystemPrompt` appends:

```
**Length — {length}:**
{length guide for that value — soft word-range target}

Length is the primary size constraint here. The mode guide above
describes register and structure within that target; if the mode's
natural length and the length target disagree, follow the length target.
```

That last sentence is load-bearing: without it, `essayistic + brief` reads contradictory and the model's resolution is unpredictable. With it, length wins for size, mode wins for shape.

The header zone reorganization replaces the always-visible mode pill with `Avançado ▾` / `Advanced ▾`. Two segmented controls inside (Mode + Length), each in its own form. Summary shows the compact label when both axes are auto, `<mode>/<length>` otherwise (e.g. `essayistic/brief`). Mobile uses the same `position: fixed` panel pattern set up earlier, with both segmented controls stacked vertically.

### CV1.E9.S6 — Soul Voice as cast-level alternative to personas

Per-turn Alma (auto via `is_self_moment` + manual via `Send to…`) didn't cover the case where the user wanted a whole stretch of conversation through Alma. The natural surface to control "who answers" was the Cast — it's already the per-session voice declaration.

Mutual exclusion is the invariant that makes the cast surface coherent: `composeAlmaPrompt` replaces the persona path, never composes with it. A session with `[mentora, terapeuta]` in the cast plus Alma forced from elsewhere is internally inconsistent. Enforcing exclusion at the data layer (transaction in `setSessionVoice` deletes `session_personas` rows when voice flips to "alma"; `addSessionPersona` via `/conversation/tag` clears voice) keeps the cast truthful.

Streaming pipeline picks up:

```ts
isAlma = forcedDestination
  ? forcedDestination.type === "alma"
  : sessionVoice === "alma" || reception.is_self_moment === true
```

The forced destination wins both directions — `Send to mentora` on an Alma-cast turn routes that turn through mentora; `Send to ◈` on a persona-cast turn routes through Alma. Reception still runs on Alma turns; `is_self_moment` becomes redundant but `org/journey/mode/is_trivial` are still meaningful for scope seeding and mode resolution.

The Cast renders a single ♔ avatar in the warm-amber Alma color (`#b8956a`) when voice is set. Glyph chosen for visual distinctness from `◇ persona` and `⌂ org` and from the previous Alma marker `◈` (which the user flagged as resembling the mirror itself). The cast `+` picker carries Soul Voice as a first-class entry at the top — its own form posting to `/conversation/voice`, distinct visual treatment, divider below before the persona select.

### Soul Voice label corrects to locale-aware

The 2026-04-27 vocabulary asymmetry decision had pinned the user-facing label to "Voz da Alma" across all locales as a brand exception, with the rationale that "alma" was universally accessible. Real use on 2026-05-01 surfaced the cost — docs reading "Voz da Alma" embedded in English prose, en-locale users encountering a Portuguese phrase mid-UI without context. The "alma is universal" rationale didn't survive the test of explaining the term in EN UX.

Returns the term to the i18n table: "Soul Voice" in en, "Voz da Alma" in pt-BR. Six i18n keys updated, SSR templates use `ts("header.cast.alma")` instead of hardcoded literals, `chat.js` reads the localized label from a `data-alma-label` attribute on `#messages` (set by SSR using the same key) so streaming bubbles + rail re-renders match the request's locale. Server-side adapters that prepend the bubble marker (sync API, Telegram) branch on `user.locale` — they don't have web's i18n middleware in scope, so a tiny inline conditional is the pragmatic shape. Reception's system prompt translates to "Soul Voice" (it's English instructions to the LLM). Internal identifiers (`is_self_moment`, `composeAlmaPrompt`, `voz-da-alma.ts` filename, `cv1-e9-voz-da-alma` story folder, `ALMA_PREAMBLE` constant name) stay — those are EN-internal codepoints not subject to user-facing localization. The `ALMA_PREAMBLE` prompt content itself stays in pt-BR (it's the user's voice; Alisson is bilingual but speaks pt-BR with the mirror).

### Mobile chat overhaul

A serious pass through the mobile chat surface, driven by Android Chrome usage on a LAN dev session:

**Bubble width.** Drop the 80% `.msg-body` cap on mobile and reduce `.content` side padding from 1.5rem to 0.75rem. The 80% cap was designed for desktop's left/right contrast between assistant and user; on mobile both columns share the viewport, so the cap was throwing space away on long replies. Assistant `.msg-body` is forced to `display: block; width: 100%; max-width: 100%` so its right edge aligns with the user bubble's (which always lands at 100% via text-align:right).

**Mode panel.** Anchor to the viewport (`position: fixed; top: 4rem; right: 0.5rem; max-width: calc(100vw - 1rem)`) instead of the small pouch element. The pouch sits inside a flex-wrap header row and can land anywhere horizontally; absolute-positioning to its right edge let the panel overflow off-screen on the left when the pouch wrapped to a left position. Stack the segmented control vertically so the panel is narrow and tappable.

**Read-more on long bubbles.** New `.bubble-content` wrapper injected by `chat.js` after markdown render; CSS clamps to 5 lines with `-webkit-line-clamp` on mobile only (clamp absent on desktop, so the button stays hidden). `setupReadMore` is idempotent, runs at init, on `addMessage` for fresh user msgs, and after streaming `done`. Fresh bubbles start expanded; reload-hydrated bubbles start collapsed.

**Olhar dentro becomes a bottom sheet.** Rail switches to `position: fixed; bottom: 0` on mobile so toggling surfaces the panel in the viewport regardless of scroll position. Previously the rail stacked below the messages in the flex-column flow — toggling looked like nothing happened. Bottom sheet fits the "inspect briefly and close" flow.

**Auth Secure cookie dynamic.** Browsers accept Secure cookies on HTTP-localhost as a special case but NOT on HTTP-LAN-IP (`192.168.x.y`, `*.local`). Without this detection, mobile-via-LAN-IP submitted login but the response cookie was silently dropped. Detection now reads `X-Forwarded-Proto` (Caddy sends it in prod) or the request URL's protocol — `secure: true` on HTTPS, `false` on HTTP. Works in localhost dev, LAN-IP dev, and proxied prod with zero config.

### Async UX for header forms

Every header form (mode, length, voice, tag, untag) intercepted at `submit` via document delegation and posted via `fetch`. The default POST → 302 → full page reload was bouncing the user to the bottom of the chat with a cleared input every time they configured anything.

- **Mode/length:** optimistic active-class toggle + summary recompute. Disclosure stays open, scroll preserved.
- **Tag/untag/voice:** post + GET the current path + DOMParser swap of `.conversation-header`. Pool data attrs synced from the new `#messages` element so badge-suppression and `ensureScopePill` stay correct without a full reload.
- **Rail toggle (`Olhar dentro`)** moved to event delegation on document so it survives the header replacement.
- **Native `form.submit()` fallback** on fetch failure preserves the no-JS path.

### Cast/scope pickers auto-submit

Picking a persona, org, or travessia from the inline `<select>` now posts immediately on `change` — the explicit "Add" button is hidden when JS is active (`body.js-on` + CSS rule). One tap per addition instead of two. The empty placeholder option's `value=""` is filtered out so cancelling a picker by reverting to placeholder doesn't fire a submit. SSR keeps the button so the no-JS fallback (and any non-rendering-CSS context) still has a working submit path.

### Chat input persists to localStorage

The risk surfaced from real use: typing a long message, then changing a header axis, then watching the input clear because the page reloaded. With every header form now async, the immediate cause is gone — but any future code path that reloads (a stray non-async form, F5, accidental back-button, browser crash) would re-introduce the data loss.

`chat.js` saves `input.value` to `localStorage[mirror.chatDraft]` on every keystroke; restored on page load when the input is otherwise empty (so SSR-provided values still win); cleared on successful `runSend`. Single global key for now — if multi-conversation parallel use surfaces, the key becomes `mirror.chatDraft.${sessionId}`.

### Bug fixes from real use

- **Scope auto-seed gate is symmetric across axes**: `decideScopeSeeding` in `server/scope-seed.ts`. The org/journey gate was previously `isFirstTurn`, strictly stronger than needed. Prod 28 Apr 2026: turn 1 was a casual greeting that yielded no scope, turn 2 classified `o-espelho + software-zen`; under the first-turn gate, the classification stamped entry meta (badges visible) but never reached the header's Scope zone. Now seeded when the corresponding pool is empty before the turn, regardless of turn number. Symmetric with persona seeding.

- **Bubble badge follows transition rule**: `decideScopeTransition` in `server/scope-transition.ts`. Previously suppressed when the pool already carried the pick; the seed-when-empty fix made junctions populate correctly which made the suppression rule fire correctly which hid the seed turn's badge — the moment scope began had no marker. Replaced with a transition rule symmetric with `newPersonasThisTurn`: show on the turn whose scope differs from the previous assistant turn's, suppress on continuation.

- **Per-turn override beats session voice**: `isAlma` resolution had session voice as a top-level disjunction without the forcedDestination guard, so `Send to mentora` on an Alma cast still routed through Alma. Restructured: `forcedDestination ? forcedDestination.type === "alma" : sessionVoice === "alma" || reception.is_self_moment === true`. A 7-test resolution matrix locks the invariant.

- **Mobile scope `+ Add` no longer spills off-screen**: each scope group occupies its own row on narrow viewports (`width: 100%` on `.header-scope-group` inside the mobile `@media`). Long pill names get `text-overflow: ellipsis` so they ellipsize inside the row instead of forcing the row wider than the parent.
