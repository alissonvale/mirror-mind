# v0.6.0 — The Admin Workspace

*18 April 2026*

**TL;DR**
- `/admin` is now a real landing surface with cards: Users · Cost · Activity · Latest release · Mirror memory · Models · System.
- `/admin/models` edits provider, model, prices, timeout, and purpose per role live — no restart, no JSON editing.
- `/admin/users` gains **delete** (cascade) and **role toggle** (admin ↔ user), both self-proof.
- `/docs` renders the project's own documentation inside the app — admin-only, nav collapsed by default for focused reading.
- **Begin again** and **Forget this conversation** on the mirror's rail give the user manual control over session boundaries. Old sessions are auto-titled in the background via a cheap model role.
- Sidebar redesign: avatar + name opens the Cognitive Map, "My Mirror" for the user's own side, "This Mirror" groups the admin surfaces.
- Vocabulary shift: "the install" → "this mirror". The admin is no longer running *an install* — they're running *this mirror*.

## Where we were

v0.5.0 gave the mirror a way to show itself to the user — the Cognitive Map made identity legible, the Context Rail made attention memory visible. But the admin had no parallel. Running the mirror meant editing `config/models.json` by hand, restarting the server to pick up changes, running SQL to promote a user, and reading the docs on GitHub in a tab next to the one you were trying to operate from. The product the admin experienced was not the same product the user experienced.

## What we did

**We opened the Admin Workspace.** CV0.E3 — originally a narrow "Install Administration" story set — grew into a real workspace. Two functions on the same surface: *seeing* (a dashboard that summarizes this mirror at a glance) and *acting* (user management, model config, docs reading). Symmetric with the Cognitive Map on the user side.

**The dashboard is the landing.** `/admin` renders six cards: Users (count + active in the last 7 days, link to the users page), Cost · last 30 days (approximate BRL total across all sessions, explicitly labeled "estimated"), Activity (sessions today and this week), Latest release (auto-detected from the releases folder, headline + link), Mirror memory (total identity layers across all users, broken by layer), Models (live role configuration with prices), and System (uptime, DB size, Node version). Cards are glances first; drill-down links are present where a deeper surface exists.

**Model configuration is live.** `config/models.json` stopped being a file read once at boot. A new `models` table in SQLite holds the three roles the mirror uses (main for responses, reception for persona routing, title for session labels). On first boot after upgrading, the migration seeds the table from the JSON; from then on the DB is the source of truth. `/admin/models` edits provider, model ID, prices, timeout, and purpose per role. Save takes effect on the next request — no restart. Revert to default reloads the shipped values for a single role. The dashboard's Models card reflects edits immediately, and so does the Cost card.

**User management got delete.** `/admin/users` already created users; it now destroys them too. Delete cascades through sessions, entries, identity layers, and telegram links in a single transaction. Role flips inline — admin ↔ user in one click. Both actions are self-proof: the admin can't delete themselves and can't demote themselves, on both the UI (no button) and the server (403 regardless of what the form posts).

**Docs got a home inside the app.** `/docs` navigates and renders the `docs/` tree. Admin-only — today's documentation is project-internal (roadmap, decisions, specs); a user manual for regular users is a separate future story. The nav is collapsed by default for focused reading; the user's preference persists per device. Every internal link — `.md` files, directory links, root-relative paths — rewrites to in-app routes so navigation stays inside the mirror. External links, anchors, and non-docs absolute paths (like `/map`) are left alone.

**The rail grew manual session control.** On the `/mirror` page, the rail footer now carries two actions: **Begin again** creates a fresh session while preserving the previous one in the DB, and **Forget this conversation** (destructive, native confirm) deletes entries and the session row. Begin again fires an async call to a new cheap `title` model role that labels the ending session — a session that used to disappear into a wall of timestamps now arrives at the future episodic-browse surface with a 3-6 word summary. The background-LLM-task pattern established here (own model role, fire-and-forget, fail-silent) is the template for the compaction and semantic memory stories ahead.

**The sidebar learned a new vocabulary.** The avatar and name at the top are now a link — clicking your face opens your Cognitive Map, which is your structure. "Mirror" became "My Mirror" to distinguish the personal from the operational. The old "Admin" section header is now "This Mirror" — the instance you operate. Admin-only sub-items live beneath it: Dashboard, Users, Models, Docs. Regular users see a minimal sidebar: brand, name+avatar, My Mirror, Logout. The menu structure honors the invariant that admin is a function of who is looking, not of what the UI claims exists.

**The word "install" retired.** Every admin-facing copy line that said "the install" now says "this mirror" — consistent with the project's vocabulary (curadoria, ressonância, alicerce, soberania, mapa) and symmetric with how users talk about their mirrors. Deployment events ("fresh installs", "npm install", "install the full environment") kept their word because those are verbs about the act, not nouns about the object.

## What comes next

CV0.E2 is closed. CV0.E3 has two stories shipped (S3, S4, S5, S1 — wait, three, four — all but S2) and a handful of radar items that are now well-scoped:

- **CV0.E3.S2 — Admin customizes adapters via the browser.** Mirrors S1 applied to `config/adapters.json`. The per-adapter prompt instructions (Telegram's short/conversational tone, web's depth, CLI's plain text) become editable without a restart.
- **CV0.E3.S6 — Operational alerts + usage tracking.** A proper `usage_log` table populated on each LLM call. Powers exact cost (replaces the dashboard's approximation) and surfaces silent failures like title generation timeouts or reception fallbacks.
- **CV0.E3.S7 — Export / Snapshot.** Admin downloads the DB; a user exports their own memory.
- **CV1.E3** — the memory epic continues. S4 (manual reset) shipped; next is S1 (automatic topic-shift detection), S2 (compaction), S3 (long-term semantic memory).
- **Episodic browse surface** — the other half of the reset story. Preserved sessions accumulate in the DB today with their titles; a future surface lists and reads them. Likely lands alongside CV1.E3.S3.

## Upgrading

### From any previous version

```bash
git pull && npm install && systemctl restart mirror-server
```

On first boot after upgrade, migrations run automatically:
- `models` table is created and seeded from `config/models.json` (your existing values).
- `sessions.title` column is added (existing sessions stay `NULL`; the future episodic browse treats `NULL` as "Untitled conversation").

No SQL required. No data loss. Existing model settings in the JSON populate the DB exactly as they were; edits made in `/admin/models` thereafter are the new source of truth.

### Additional steps if upgrading from v0.1.0

- Re-import identity: `npx tsx server/admin.ts identity import <name> --from-poc`
- Remove `LLM_MODEL` from your `.env` (models are configured in the DB, seeded from `config/models.json`)

### Notes for admins

- `/admin/users/:name` redirects to `/map/:name` (from v0.5.0). The user-management page is `/admin/users`.
- `/admin/models` is the new canonical surface for model tuning. `config/models.json` stays in the repo as seed + reference, but editing it after first boot has no effect on a running install — use the UI.
