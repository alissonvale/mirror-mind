# v0.10.0 — A Place to Land

*22 April 2026*

**TL;DR**
- **New entry point.** Login no longer drops you into chat. `/` is now a **home page** with greeting, *State of the mirror* (admin), *Latest from the mirror* (release digest), and *Continue* (active session + earlier threads). `/mirror` and `/chat` redirect to `/conversation`.
- **Sidebar reorganized around three questions.** *Conversation* (what I'm doing right now) at the top, *Admin Workspace* collapsed to one link, then context links grouped by **Who Am I** (Psyche Map), **What I'm Doing** (Journeys), **Where I Work** (Organizations). The avatar in the footer opens the new About You page; the Cognitive Map is renamed Psyche Map.
- **About You at `/me`.** Clerical concerns (name editing, currency preference, light self-portrait stats, data export placeholder) separate from structural identity — the operational *you* lives apart from the structural *you* on the Psyche Map.
- **Single-currency cost display.** Cost surfaces stop showing USD and BRL side-by-side. The admin picks one in `/me` (USD or BRL); every cost number across the app respects the choice. `users.show_brl_conversion` reinterprets from "show BRL alongside" to "prefer BRL over USD" — no migration.
- **Last conversation per scope.** Cards on `/organizations` and `/journeys` pair each scope with a "Last conversation" readout (title + relative time of the most recent session tagged with that scope). Zero schema change — reads existing `_organization` / `_journey` meta on assistant messages.
- **Sessions hold many voices.** [CV1.E4.S4 — Manual session scope tagging](../project/roadmap/cv1-depth/cv1-e4-journey-map/cv1-e4-s4-manual-scope-tagging/). Each session now carries a **pool** of personas, organizations, and journeys; reception picks within the pool each turn; the user can curate the pool from the Context Rail at any time. First turn of a fresh session auto-populates the pool from reception's picks ("sugeridas"). Persona stays singular per reply (one voice); orgs and journeys compose multi into the prompt.
- **376 tests passing** (was 311). Three new junction tables landed via additive migration — no SQL required.

## Where we were

v0.9.0 closed with cost visibility (budget) and subscription-OAuth infrastructure. The mirror could speak in many voices, in many scopes, and the admin could see what each call cost. But the experience after login still dropped you straight into a chat surface. Navigation was busy with admin links scattered across the sidebar. And every session had to re-derive its own context every turn — reception's per-message signal was the only anchor; if reception missed, there was no way to correct.

This release reshapes the entry path and gives sessions explicit context they carry across turns.

## What we did

### CV0.E4 — Home & Navigation (the full epic, S1–S7)

**S1 — Landing home (`/`).** New authenticated route. Greeting, *State of the mirror* card (admin only — users / cost / activity / system), *Latest from the mirror* (digest of the most recent release; existing release files retroactively gained `digest:` frontmatter), *Continue* band (active session + up to 3 earlier threads). Login redirects here instead of `/conversation`.

**S2 — Sidebar pruning + admin shortcuts.** The *This Mirror* section collapsed from six sub-links into a single **Admin Workspace** link. `/admin` itself becomes the navigation hub with shortcut cards for the five admin surfaces (Users, Budget, Models, OAuth, Docs). The dashboard's stale "Cost" card was replaced by a real-data Budget card.

**S3 — Three-questions sidebar.** Context links restructured into three labeled sections: *Who Am I* (Psyche Map), *What I'm Doing* (Journeys), *Where I Work* (Organizations). **Cognitive Map renamed Psyche Map** throughout — promoted to a first-class link from any chat surface. *Conversation* stays as the top primary action.

**S4 — About You (`/me`).** New route absorbs clerical concerns: name editing, display preferences (currency choice landed here in S6), light self-portrait stats (most active persona, session count), data export placeholder. Clicking the avatar in the sidebar now opens `/me` instead of the Psyche Map. Operational *you* and structural *you* live separately.

**S5 — URL alignment.** `/mirror` and `/mirror/*` rename to `/conversation` and `/conversation/*` to match the sidebar label that S3 introduced. `/mirror` and `/chat` both 302 to `/conversation` for backward compatibility — bookmarks survive.

**S6 — Single-currency cost display.** `/admin/budget` no longer renders USD and BRL side by side. The admin picks one in `/me` preferences via two radios (USD / BRL); every cost surface across the app respects the choice. `formatUsdAndMaybeBrl` removed; replaced by `formatCost(usd, rate, preferBrl)` that returns one currency. `users.show_brl_conversion` semantically reinterprets from "show BRL alongside USD" to "prefer BRL over USD" — every existing user's experience preserved.

**S7 — Last conversation per scope.** `/organizations` and `/journeys` list pages pair each scope card with a "Last conversation" card showing the title + relative time of the most recent session tagged with that scope. Window-function SQL (`ROW_NUMBER() OVER PARTITION BY json_extract(...)`) over existing `_organization` / `_journey` meta on assistant messages — no schema change. Adapter coverage caveat: scopes touched only via Telegram or API show "No conversations tagged yet" because those adapters stamp only `_persona`.

### CV1.E4.S4 — Manual session scope tagging

The session↔scope relationship flips from 1:N (one persona / one org / one journey per turn, implicit via assistant message meta) to **N:N explicit** — three new junction tables (`session_personas`, `session_organizations`, `session_journeys`) — so the user can curate the pool of contexts an entire conversation operates within. Derived from a product-design conversation where it became clear that **perfect reception can't be guaranteed** and manual override needed to be first-class.

**Hybrid model:**
- The session declares a **pool** of personas / organizations / journeys.
- Reception picks **within** the pool each turn (empty pool = considers all candidates, backward-compatible).
- The user can **correct** the pool from the Context Rail at any time — pills with × to remove, dropdowns to add.
- **First turn** of a fresh session auto-populates the pool from reception's picks ("sugeridas" default).
- **Persona stays singular** per reply (the mirror has one voice per turn). Orgs and journeys compose multi into the prompt — broader before narrower, all tagged scopes injected.

**Implementation across four phases:**
1. Schema + helpers — three junction tables with composite PK, string keys (consistent with reception output). `getSessionTags` + add/remove per type + `clearSessionTags`. `forgetSession` cascades.
2. Reception filtering — `ReceptionContext.sessionTags` narrows the candidate set before the LLM call. Reception output stays the same shape; the candidate set is what shrinks.
3. Composer multi-scope — `ComposeScopes.sessionTags` renders all tagged orgs and all tagged journeys; persona stays singular. Backward fall-back to reception's single pick when a type has no tags.
4. Rail UI + endpoints — new "Scope of this conversation" section on the Context Rail with three tag groups. Each group: pills (× removes via `POST /conversation/untag`) + dropdown-add (`POST /conversation/tag`). RailState gains `tags` with candidate lists.

**First-turn suggestion** — `/conversation/stream` detects `entries.count === 0 && totalTags === 0` before reception runs, and if so writes reception's non-null picks into the session tags before composing the prompt. The next turn already operates within the newly-seeded pool.

## Upgrading

### From v0.9.0

```bash
git pull
npm install
systemctl restart mirror-server
```

On first boot after upgrade, `migrate()` runs additive changes:

- **`session_personas`**, **`session_organizations`**, **`session_journeys`** tables created (CV1.E4.S4). Empty until the user tags a session from the rail. Existing sessions continue to operate via the per-message meta, backward-compatible.

`users.show_brl_conversion` is **reinterpreted, not migrated**. The column name stays; its meaning shifts from "show BRL alongside USD" to "prefer BRL over USD". Toggled-on users see BRL only (small noise reduction); toggled-off users see USD only (unchanged). One-line comment at each read site notes the historical artifact.

No SQL required. No data loss.

### Recommended (manual, after upgrade)

1. **Visit `/`** to see the new home. Login redirects here now; if you bookmarked `/mirror`, the redirect to `/conversation` carries you forward.
2. **Visit `/me`** to set your currency preference (USD or BRL) — the budget page and the rail will respect it from the next page load.
3. **Open `/conversation` and look at the rail.** The new "Scope of this conversation" section is below the persona; first turn of a fresh session will auto-populate it after your first message.
4. **Open `/organizations` or `/journeys`** to see the "Last conversation" readouts — they populate retroactively from existing assistant-message meta.
