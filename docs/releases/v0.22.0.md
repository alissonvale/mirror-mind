## What we did

### CV1.E11 — Scenes (the cena pivot epic)

The pivot insight: a **scene** is not a feature on top of the existing model. The scene IS the model. Personas, organizations, journeys are consequences that emerge in service of specific scenes — they are not pre-requisites the user assembles upfront. The seven stories trace the full implementation from data layer to user-visible surface to onboarding to cutover.

**S4 — Backend (foundational).** New `scenes` table with the asymmetric cardinality the design called for: cast as a junction (`scene_personas`, mutable, ordered, multi-element); scope as single-key columns (`organization_key`, `journey_key`, stable, single-element). The asymmetry mirrors the conversation header redesign in CV1.E7.S2 — personas form a *cast* (mutable ensemble), orgs and journeys are *scope* (stable context). `sessions.scene_id` links a session to a cena via `ON DELETE SET NULL` semantics — deleting a cena unscopes its conversations rather than destroying them.

`findMatchingScene(db, userId, receptor)` implements the locked-design strict matrix:
- **Alma cena:** `is_self_moment=true` AND `scene.voice='alma'` (precedence over persona match)
- **Persona cena:** `receptor.personas[0] ∈ scene.cast` AND `scene.organization_key === receptor.organization` (or both null) AND `scene.journey_key === receptor.journey` (or both null)
- Multiple matches → most-recent activity wins
- Trivial turn → null up front

The strict-match matrix keeps cold-start suggestions credible while CV1.E8.S1 telemetry accrues; looser matching (org alone, semantic similarity over briefing) is deferred until enough calibration data exists to know which loosenings are safe.

**S7 — Cena form (inline expander, stub-first).** Dedicated routes `/cenas/nova` (create) and `/cenas/<key>/editar` (edit) — sharing a single form component. Title is the only required field; briefing is the dominant visual element (180px min-height, the substance the LLM reads every turn). Voice toggle is mutex with cast at both the data layer (CV1.E11.S4) and the JS layer (flipping to Alma hides the Elenco fieldset and clears the personas input as a defensive measure). Dual save buttons: `[Salvar]` returns to home; `[Salvar e iniciar conversa]` chains create-cena → create-session → redirect to `/conversation/<sessId>` in one shot.

Stub-first inline sub-creation in P4: typing a name in Elenco / Organização / Travessia that doesn't match an existing entity surfaces `+ Criar X "name"` in the autocomplete dropdown. Click expands a mini-form (name, key auto-derived but editable, optional description); saving POSTs JSON to `/cenas/sub/<kind>` which creates the entity with `is_draft=1`. The new entity's key gets injected back into the cena form's field; the inventory updates locally so subsequent typing finds it. Stub creations are committed (not transactional with the cena form) — undoing creation surprises; the "Salvar como rascunho" label makes the commit explicit.

`is_draft INTEGER NOT NULL DEFAULT 0` columns added to `identity`, `organizations`, `journeys`. Persona-only at the helper layer (other layers ignore the flag). Promote-on-edit: opening a draft entity's workshop and saving (even with no field changes) flips `is_draft=0` — the workshop save is itself a deliberate review act.

**S1 + S2 — Home + chrome (the user-visible payoff).** AvatarTopBar component is the chrome for the new surface family — logo + brand on the left, avatar with dropdown on the right. Dropdown items: name + email header (clickable → /me), Mapa Cognitivo (/map), Minha Memória (/memorias — placeholder until S3), Skills (em-breve), Admin + Docs (admin only), Sair (POST /logout). No center navigation — the sidebar's seven equal entries are demoted to this single menu.

`InicioPage` (Variante C from the locked design): cena cards above with `✚ Nova cena` card at the end, "ou" separator, free input below, recents below. Each card is a `<form method="POST">` wrapping a button styled as a card (so click POSTs to `/cenas/:key/start`). Free input redirects to `/conversation/<sessId>?seed=<text>`; `chat.js` consumes the `seed` query param and auto-sends via the existing message API on load.

Briefing-in-compose (P4 of S1+S2): `composeSystemPrompt` and `composeAlmaPrompt` accept optional `scene` parameter. When non-null, renders `## Cena: <title>\n\n<briefing>` between the persona cluster and the org/journey scopes — the cena describes "this specific recurring conversation pattern", narrower than identity, broader than situational scope. Empty briefing still renders the header so the model knows a cena is active. All three adapters wired (web stream, web sync, telegram) — each loads the scene via `session.scene_id` before composing.

Cold-start (P5 of S1+S2): `evaluateColdStart(db, userId, sessionSceneId, isFirstTurn, reception)` gates on (unscoped session, turn 1, non-trivial) before calling `findMatchingScene`. The web stream attaches `cenaSuggestion: {key, title, glyph}` to the SSE "done" event payload. `chat.js` renders the suggestion card below the assistant bubble; accept POSTs to `/conversation/:sessId/apply-scene` and reloads to update the header. Cena applies from turn 2 onward — turn 1 is not re-run (re-running burns tokens, confuses the user, and the baseline answer is fine).

**S3 — Memórias dashboard.** Second TopBarLayout surface. 2×2 grid (Cenas, Travessias, Organizações, Library) plus Histórico full-width below. Each card: count + 3 most-recent items + "ver →" link. Library is a deliberate honest placeholder (em-breve badge + body text "atalhos para documentos e materiais que viram contexto. Em construção.") — the slot stays so the 2×2 shape lands; CV1.E4.S2 will fill it. Histórico mirrors `.conversations-rows` markup from `/conversations` so the visual is consistent. Capped at 20 most recent sessions; "ver tudo →" links to `/conversations`. Bonus deliverable: `/cenas` simple list page so the Cenas card's "ver →" has somewhere to land.

**S6 — Onboarding seed.** `handleUserAdd` in `server/admin.ts` refactored to extract a pure `provisionUser` helper. New users get four seeds: ego/behavior (CV0 baseline), ego/expression (CV0 baseline), self/doctrine (from `docs/seed/alisson/doctrine.md` — Alisson's 9 Princípios as the v1 default for the household phase), and a Voz da Alma cena (voice='alma', empty briefing — the cena's role is to give the new tenant a Voz da Alma entry point on /inicio from day one; the Alma compose path supplies substance via soul + doctrine). `self/soul` intentionally NOT seeded — the Mapa Cognitivo's "create the layer" invitation surfaces on first use. Defensive read on the doctrine seed file — missing file logged + skipped without failing user creation. Narrative-loader applies the Voz da Alma seed (idempotent) to existing narrative tenants.

**S5 — Cutover (small, then formalized).** Initial cutover (`web.get("/")` → 302 to `/inicio`) shipped as the small PR the design asked for. Subsequent formalization (request from the user this session) made the cutover real: Layout component rewritten in place to render AvatarTopBar instead of sidebar (one file change, all pages migrate). HomePage component retired. Sidebar JS / CSS / i18n keys deleted. The strangler chrome split is over — the avatar bar is the canonical chrome.

### Brand redesign

Logo: `⌘` (Place of Interest) → `◆` (Black Diamond). Label: localized i18n key (`topbar.brand`) → hard-coded "Mirror Mind" (locale-independent — it's a product name, not a translatable noun). The diamond reads as the mirror metaphor (refraction, multiple faces).
