# v0.4.0 — The Web Grows Up

*16 April 2026*

**TL;DR**
- Web client moved to `adapters/web/` — same pattern as CLI and Telegram
- Sidebar navigation replaces top nav
- Chat gets visual identity — warm palette, persona badges, markdown rendering
- 13 new web route tests (68 total)
- Docs restructured: roadmap hierarchy with named CVs, epics, and stories
- `design/` renamed to `product/`

## Where we were

v0.3.x delivered adapter awareness — the mirror adapted its tone per channel and formatted output for Telegram. But the web client was a patchwork: pages scattered inside `server/`, admin split across separate screens, a flat top nav with no hierarchy, and a chat that looked like any chatbot. The codebase didn't reflect its own architecture — clients are adapters, but the web wasn't in `adapters/`.

## What we did

**We gave the web its own home.** All web pages, routes, auth, and static assets moved from `server/` to `adapters/web/`. A `setupWeb(app, db)` function follows the exact pattern of the Telegram adapter. The server became core-only — 120 lines, down from 360. The folder structure now says what the architecture says: server is the core, adapters are the edges.

**We added a sidebar.** The top nav was flat — Chat and Users at the same level, no hierarchy. The sidebar organizes: Mirror Mind brand at the top, Chat as the main action, Admin as a section, Logout at the bottom. On mobile it collapses behind a hamburger. The login page stays clean — no sidebar, no chrome.

**We gave the chat an identity.** The message area went from grey to warm cream. Assistant bubbles gained subtle warm borders. The persona signature became a pill badge — small, muted, above the bubble. And the LLM's markdown output now renders as real HTML in the browser: headers, bold, code blocks, lists.

**We added web route tests.** 13 tests using Hono's `app.request()` — login flow, cookie auth, admin pages. No real server, no browser needed. :memory: SQLite, in-process HTTP simulation.

**We restructured the docs.** The documentation now mirrors the roadmap literally: `docs/project/roadmap/cv0-foundation/cv0-e1-tracer-bullet/cv0-e1-s1-db-identity/`. Each folder with subfolders has a navigation index. `design/` became `product/` — because that's what it is.

## What comes next

The web is mature. But the mirror has a problem: when you change subject mid-conversation, the old context bleeds into the new one. Long conversations accumulate history that the LLM can't filter.

The next step is **topic shift detection**. The reception layer — which already classifies every message for persona routing — will gain awareness of recent history. When it detects a subject change, the server silently creates a new session. No "new conversation" button. No command. The mirror decides for itself that the context has changed.

After that: compaction (summarizing old history instead of discarding it), long-term memory (remembering across sessions), and journeys (knowing what the user is navigating right now).

## Upgrading

### From any previous version

```bash
git pull && npm install && systemctl restart mirror-server
```

### Additional steps if upgrading from v0.1.0

- **Re-import your identity** to include personas: `npx tsx server/admin.ts identity import <name> --from-poc`
- **Remove `LLM_MODEL` from your `.env`** — models are now configured in `config/models.json`
