## Where we were

v0.23.0 (*The mirror has a face*) shipped the contemplative entry-point at `/` — the espelho as a single self-portrait synthesizing Vivo / Estou / Sou. The typographic system established there (EB Garamond italic, axis colors, numeric tiles, drop-cap on Sou) became the typographic vocabulary v0.24.0 extends across every entity.

After v0.23.0, the user landed on a journey workshop page (`/journeys/bia-saturada`) and noticed the friction: `/espelho` invites contemplation; the workshop demands editing. Same matter, opposite metabolism. The product-designer conversation that opened v0.24.0 named the tension and proposed the fix — every entity link should land on a memoir-shaped read view, with the form behind a discreet door.

## What we did

### CV1.E13 — Portraits epic (read view by default for entities)

The design conversation across one session authored three reference drafts for Antonio's travessias (Bia Saturada, Voltar a BH, Pós-Lançamento) and consolidated ten principles before any code: sections render conditionally on data, lede length is variable, absences are declared not silent, structural patterns in the source produce structural sections, briefing-sourced fechos read with more weight than conversation-sourced, etc.

The implementation across S1-S4 trades reuse for variation: ~70-80% of the engine carries between stories (the URL migration framework, the typed portrait state, the section components in `portrait-shared.tsx`, the citable-line extractor with `entity_profile_cache`), while each story adds the editorial sections specific to its entity type.

S1's reference-draft test (`tests/journey-portrait-reference-drafts.test.ts`) seeds Antonio's verbatim source content into a fresh DB, computes the portrait, and asserts the structural shape — Bia Saturada's lede comes from briefing's `descuido` paragraph, Voltar a BH's structural section emits 3 cenários, Pós-Lançamento's "Saí, mas eles não desapareceram" lands as the close. The drafts are the acceptance bar.

S2's lede heuristic flips to **situation-first** because org briefings tend to be identity manifestos ("Pages Inteiras é minha casa editorial..."); the diagnosis of *what's alive right now* lives in the situation field. The picker takes situation's first paragraph and optionally appends a short briefing-end punchline when both fit on a comfortable two-line read.

S3's voice bifurcation handles two qualitatively different cenas: persona-voiced cenas list the cast personas with their descriptors via the `scene_personas` junction; alma-voiced cenas render a single ♔ Voz da Alma indicator with the warm-amber glyph from the espelho. The same shell stays plum (`#9a8ba0`); only the cast section's content varies.

S4's per-row accent color is a first in the epic — each persona has its own `identity.color` already used across the system (avatar chip, bubble bar, /map card); the portrait completes the coherence by adopting that same color for its left strip and tile numbers. Scoped via `[data-entity="persona"]` so other portraits keep their axis-fixed accents. The two new editorial sections (POSTURA + ANTI-PADRÕES) extract from the authored markdown headings — Antonio's `marido` persona has 3 paragraphs of POSTURA and 6 lines of ANTI-PADRÕES, both rendered with the persona's color leading the chrome.

The URL migration is consistent across S1-S3: `GET /<entity>/<key>` → portrait, `GET /<entity>/<key>/{editar,edit}` → workshop. S4 takes a more conservative path — the legacy `/map/persona/<key>` URL is preserved as the workshop because internal links across the codebase already point there; the portrait URL `/personas/<key>` is new, and `/personas/<key>/{editar,edit}` aliases redirect to the legacy URL for cross-locale link sharing.

### CV1.E14 — Identidade (the metaphor change)

After CV1.E13 made every entity surface a read-view-by-default, the cognitive map (`/map`) became conspicuous as the last surface still presenting the user's psyche as a structural grid. The user named it: *"the map metaphor doesn't match how identity reads. Identity isn't a grid; it's a story."*

The story shipped in two passes:

1. First pass: page named **`/narrativa`** with five flat sections ALMA / IDENTIDADE / COMPORTAMENTO / EXPRESSÃO / ELENCO. The section labeled "IDENTIDADE" carried the `ego/identity` layer's content — what I do, how I introduce myself, who I am not.

2. Second pass (same session, immediately after first): the page name + section label collision felt redundant. **Page renamed to `/identidade`** to match the avatar dropdown's natural label; **section relabeled to `PAPEL` / `ROLE`** so the section that holds the `ego/identity` layer doesn't collide with the page name. Mechanical rename touched route URLs, file names (`narrativa-synthesis.ts` → `identidade-synthesis.ts`, `narrativa.tsx` → `identidade.tsx`), function/type names (`composeNarrativa` → `composeIdentidade`, `NarrativaState` → `IdentidadeState`), state field naming (`state.identidade` → `state.papel`), CSS class prefixes (`.narrativa-*` → `.identidade-*`), i18n key namespace (`narrativa.*` → `identidade.*` with `.section.identidade`/`.stub.identidade` → `.section.papel`/`.stub.papel`), and chrome links across `/espelho` Sou pane + avatar dropdown.

The synthesis is **light** — no LLM. `parseLayer(content, editPath)` is a pure helper: strips the leading `# H1` (already redundant with the section label rendered by the page), captures preamble paragraphs before the first `## ` heading, splits the rest into named subsections by `## ` markers. Returns `isEmpty: true` when the content has no body at all; the page renders an italic stub block with an inline edit link in that case ("Esta camada ainda não foi escrita — escrever").

Sub-headings within each layer render as `☞ heading` in italic serif. `☞` (BLACK INDEX POINTING RIGHT) is the typographic finger; reads as orientation into the next chapter, not as an `<h3>` that breaks the page into cards. The full layer markdown is rendered verbatim with paragraph breaks preserved.

The **"ego" term retires from the user-facing chrome** in this story. The DB schema (`identity.layer = 'ego'`) and internal code (system prompt composition, route handlers) keep "ego" as architectural nomenclature. A follow-up audit will sweep remaining "ego" labels in the legacy `/map` workshop home, layer breadcrumbs (`← Mapa Interior`), and layer-workshop chrome.

### `/espelho` polish

Five smaller passes during v0.24.0:

1. **Name bookplate** at the top of `/espelho` — small caps, 0.32em letter-spacing, muted gray (`#a0aec0`), centered, with a 3rem horizontal rule below. Reads as a placa: the user knows whose reflection they're looking at without the page tipping into "profile" register. Renders even when the synthesis is empty (fresh user) so the page always carries the anchor.

2. **Pane copy flip** — title and subtitle exchange roles. Was: title = verb ("Sou", "Estou", "Vivo"), subtitle = question ("quem sou?", "por onde opero?", "o que estou vivendo?"). Now: title = question, subtitle = descriptive placa ("A voz e os princípios que me organizam por dentro" / "Travessias e cenas que estruturam meus dias" / "Vozes e conversas que preenchem meu dia"). PaneHeading prop renamed `question` → `subtitle`; CSS class `.espelho-pane-question` → `.espelho-pane-subtitle`; i18n keys `espelho.depth.<axis>.heading` → `.title` and `.question` → `.subtitle`.

3. **Vivo glyph swap** from `◌` (DOTTED CIRCLE — almost invisible next to its sibling glyphs `✦` Sou and `◉` Estou) to `✼` (six-petal asterisk). Matches the asterisk-shape family of `✦` without competing.

4. **`soul-updated` shift retires.** "Camada da alma atualizada hoje." leaked structural language ("camada", "alma" as system layer) into a contemplative surface and told the user something they already knew (they edited their own soul). Marker type, synthesis branch, ShiftItem case, and three i18n keys per locale all stripped. Existing test probes flipped to assert the marker is gone (defensive).

5. **`many-conversations` threshold raises from `> 0` to `>= 3`.** "1 conversa nova desde a última visita" used to fire when the user came back to `/espelho` 20 minutes after their last session. The marker now reads as "you've been busy" instead of every casual return triggering a tap on the shoulder.

### Look-inside rail surfaces the cena

Bug pending since CV1.E11.S1: the rail's Composed block listed persona / organization / journey / mode but silently dropped the cena, even though `composedSnapshot.scene` had been part of the type since CV1.E11.S1. Two gaps: (1) `buildRailState` never read `sessions.scene_id` into the snapshot — `composedSnapshot()` was called without `sceneKey`, so canonical sessions anchored to a cena rendered with `composed.scene = null`; (2) `context-rail.tsx` had no `rail-composed-scene` row at all. Fix wires the session's scene through to the snapshot and renders a row between persona and organization (mirrors the actual prompt order — cena injects between persona and org/journey via `briefing-in-compose`). New i18n key `rail.sceneLine` ("scene: {key}" / "cena: {key}"), live-update logic in `chat.js`, asset bump to `chat.js?v=look-inside-scene-1`.

### Narrative tenants enriched

Six tenants (Antonio Castro, Bia Lima, Dan Reilly, Elena Marchetti, Eli Reilly, Nora Reilly) get:

- A `## Summary` section in each `identity/self/soul.md` — one distilled line in the tenant's voice, ≤200 chars. The narrative loader extracts it and persists via `setIdentitySummary` so `/espelho`'s Sou pane reads a clean line instead of `pickSummaryOrFirstSentence` extracting from the layer body (which previously caused "# Soul ## Quem eu sou Eu sou alguém..." to render as if the H1 + H2 markers were prose).

- A new `<userDir>/inscriptions.md` with 3-5 phrases per tenant, mixing the tenant's own principles with citations from their authors of reference. One inscription marked `*(pinned)*` per tenant so the `/espelho` top tile lands on a stable line.

- Each tenant's `index.md` updated to reference `inscriptions.md`.

Loader plumbing (`server/import/narrative-loader.ts`):

- `upsertIdentity` extracts `## Summary` from any identity layer file and persists via `setIdentitySummary`.
- New `loadInscriptions` parses bullet-list lines (`- "text" — Author *(pinned)*`) and seeds with `createInscription` + `pinInscription`. Idempotent on the user — re-runs leave existing inscriptions alone.
- **Self-attribution stripping** added later: when a parsed inscription's `author` matches the tenant's own name (case-insensitive, trimmed), the row is created with `author = null`. The markdown stays self-documenting (you know whose line is which); the runtime behavior is what changes — the bookplate at the top of `/espelho` already names the user, so persisting the same name as `author` on a self-quote rendered redundantly under the tile.

   ─

   ◆  Espelho       → /espelho

   ◌  Memórias      → /memorias

Visual notes:

- The `◆` glyph for Espelho **reuses the brand mark** — reinforces "this is the doorway to the mirror" the logo already carries, just textually.
- The `✦` glyph for Identidade **reuses the Sou pane glyph** from `/espelho` — surfaces speak the same language across pages.
- `⌂` for Território is borrowed from the rail's organization glyph — territory contains orgs (and journeys + scenes), so the home-shape reads correct as a parent.
- `◌` for Memórias picks up the dotted-circle that used to be Vivo's glyph (pre-`✼`) — fits memory as residual trace.

The decision to stay with Unicode glyphs over an icon-font library (Lucide / Heroicons / Feather) was deliberate: the system's typographic vocabulary was chosen for literary tone, not generic SaaS convention. For ~9 menu items, Unicode delivers without a new dependency, build step, or font-file fallback.

CSS adjustments: `.avatar-top-bar-dropdown-glyph` cell at fixed 1rem width with muted color; `.avatar-top-bar-dropdown-item` flips from `space-between` to `gap: 0.7rem` so glyph + label sit naturally flush-left; `.avatar-top-bar-badge` gets `margin-left: auto` to push the "em breve" badge to the right edge without depending on flex justification.
