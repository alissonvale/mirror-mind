# v0.1.0 — The Tracer Bullet

*13 April 2026*

**TL;DR**
- Mirror server running 24/7 at `mirror.softwarezen.com.br` with HTTPS
- Identity in layers (soul, ego/identity, ego/behavior) — Jungian architecture, imported from POC
- Three channels, one conversation: CLI, Web (with SSE streaming + admin), Telegram
- Bearer token auth, SQLite persistence, Agent per request via pi-agent-core
- Admin CLI for user provisioning and identity management
- Documentation as wiki — roadmap, principles, story docs, decisions log

## Where we were

Mirror Mind existed as an idea validated by a Python PoC — a CLI tool coupled to Claude Code that worked for one person on one machine. It had 13 personas, RAG memory, journey tracking, and a Jungian identity architecture. But it was trapped: single-user, single-device, single-session. Nobody else could use it. Conversations didn't travel. Closing the terminal meant starting over.

A spike session with the group (Henrique, Alê, Vini) validated pi as the foundation for a reconstruction. The decision was greenfield: same soul, new body. Client-server from day one.

## What we did

We fired a tracer bullet — the thinnest possible path through the entire system, end to end.

In one day, six stories went from zero to running in production:

**The mirror got its voice.** A SQLite database with identity stored in layers (soul, ego/identity, ego/behavior), imported from the PoC. Not a flat text blob — structured, editable, extensible. Each layer has meaning in the Jungian architecture.

**The server came alive.** A hono HTTP server with bearer token auth, system prompt composed from identity layers, and a pi-agent-core Agent instantiated per request. One POST, one reply, one persistent thread.

**It went to the cloud.** Deployed on a VPS with systemd and nginx, accessible at `mirror.softwarezen.com.br`. HTTPS via Cloudflare.

**Three channels, one conversation.** A CLI for the terminal, a web UI with real-time streaming (SSE) and admin pages, and a Telegram bot — all hitting the same server, the same session, the same identity. Send a message on Telegram, see it in the web. The thread is one.

Along the way, we established a development process: design before code, small stories with immediate validation, living documentation, refactoring as part of the cycle. The docs are a navigable wiki that mirrors the roadmap hierarchy.

## What comes next

The tracer bullet proved the architecture works. The mirror exists, it speaks, it persists, it's accessible. But it speaks with one voice — the base identity, without nuance. Every reply has the same flavor regardless of whether the question is about therapy, code, or strategy.

The next step is **depth**: personas that respond with the right lens for each context, memory that carries knowledge across conversations, and journey context that knows what you're navigating right now. The mirror should feel alive — not just present, but aware.
