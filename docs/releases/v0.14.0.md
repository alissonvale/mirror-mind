# v0.14.0 — Finding the voice

*24 April 2026*

**TL;DR**

- **[CV1.E7 — Response Intelligence](../project/roadmap/cv1-depth/cv1-e7-response-intelligence/) activates.** New epic whose tracer bullet, [S1 — Expression as a post-generation pass](../project/roadmap/cv1-depth/cv1-e7-response-intelligence/cv1-e7-s1-expression-pass/), moves `ego/expression` out of the composed system prompt and into a dedicated second LLM call that reshapes the draft. Reception gains a fourth axis (response mode — `conversational` / `compositional` / `essayistic`). Two-phase streaming UX in the web chat: *Composing…* while the draft generates (hidden from UI), *Finding the voice…* while the expression pass runs, then the final text streams. New `expression` model role (Gemini 2.5 Flash, 10s timeout) in `config/models.json` auto-inserted on upgrade. `sessions.response_mode` column lets the user override the mode per-session via the Context Rail.
- **[CV1.E7.S2 — Conversation header + slim rail](../project/roadmap/cv1-depth/cv1-e7-response-intelligence/cv1-e7-s2-conversation-header/)** redesigns `/conversation` around the asymmetry between personas (cast) and scope (context). New `<ConversationHeader>` above the chat: Cast zone (avatars of personas in the pool, click to see descriptor + turn count + dismiss, `+` to convoke), Context zone (org `◈` and journey `≡` pills, overflow collapses to `+N more`), Mode pill (click expands a segmented control, click-to-commit), and a `⋯` menu (New topic / Look inside / Forget). The old rail-as-junk-drawer collapsed: the rail is now **admin-only**, hidden by default, toggled via the menu's Look inside item. Message bubbles gain a **persona signature** — a 3px color bar on every persona'd assistant bubble, plus a `◇ persona` text badge that renders only on persona transitions.
- **Persona colors improvement.** Each persona now carries a persistent, editable color stored in `identity.color`. A native color picker on `/map/persona/<key>` lets the user choose any color; the picked color propagates live to every surface where the persona appears — Cast avatars in the header, bubble color bar + `◇ persona` badge, `/conversations` persona tags, Psyche Map persona card, `/personas` listing avatar, and the streaming `routing` SSE event. Migration backfills existing personas with their hash-derived color so the upgrade is visually invisible until the user edits.
- **Refinements pulled from real use** — title on first turn (not only on Begin again); per-row title-regenerate `↻` on `/conversations`; delete-turn `×` on every message bubble (user + assistant, one confirm, ownership-checked); sidebar `New` entry between Current and See All; rename *Begin again* → *New topic* and *Scope* → *Context* (labels only; routes and internals unchanged); popovers close on click-outside; rail POSTs respect the session the user is currently viewing; reception prompt recalibrated so short first-person statements about deep topics classify as conversational (not essayistic).
- **627 tests passing**, up from 513 at v0.13.0 (+114). Twenty-one test files, five new describe blocks.

## Where we were

v0.13.0 (*The Family Moves In*) populated the empty system with four fictional users. Authoring their conversations in prose exposed the shape real use would take: short, lived-in exchanges (*"Had coffee with Mike Fraser this morning."*), answered with unapologetically long, structured essays because `ego/expression` was competing with `self/soul`, persona, and scopes inside a single mega-prompt. Form rules had no dedicated pass; the model couldn't be short when the substance rules wanted depth, and vice versa.

Separately, the rail had accumulated seven blocks of equal visual weight — a junk drawer. Information (Composed, Session stats) and action (Scope tags, Response mode, New topic, Forget) intercalated with no hierarchy. The chat itself was a neutral cream column in the middle, with no visible sign of *who* was speaking other than a small text badge above each assistant bubble.

This release is the response to both pressures in one coherent move.

## What we did

### Response intelligence moves from prompt to pipeline (CV1.E7)

A new epic codifies a premise the briefing already named (*#5 — every token in the prompt must earn its place*): stop putting every signal into one system prompt and start composing intelligence across named pipeline steps. S1 peels off the smallest, clearest signal first.

**S1 — Expression as a post-generation pass.** Every chat turn now passes through two LLM calls: the Agent produces a draft (the substance), then a dedicated second call (the `expression` role) reshapes it (the form). The expression pass receives the draft, the user's original message, the active persona, the user's `ego/expression` content, and the mode — and returns the final text. Substance doesn't compete with form for weight any more; each gets a model sized for its job.

The response mode — classified by reception as a fourth axis alongside persona / organization / journey — picks among three shapes:
- **conversational**: short, lived-in replies. Default when the lighter path applies. Short first-person statements stay conversational even when the topic is existential or developmental.
- **compositional**: structured, list-shaped, how-to.
- **essayistic**: reflective, prose-over-lists, explicit asks for depth.

The user can override the mode per-session via the Context Rail (then: header) — the override wins over reception's pick for that whole conversation.

The web chat shows two microtext phases during generation so the user sees what the mirror is doing between send and reply:
1. `Composing…` while the Agent writes the draft (hidden — the direct-stream is the cost of the pass).
2. `Finding the voice…` while the expression call starts.
3. The final text streams word-by-word as the expression pass emits it.

New `expression` model role auto-inserts on boot via `addMissingModelRoles()` — the DB-side migration is idempotent and doesn't touch admin customizations of existing roles.

### Cast-as-ensemble UI (CV1.E7.S2)

A product-designer review of the rail-heavy `/conversation` flagged seven blocks of equal weight and an asymmetry-blind treatment of the three axes (persona / org / journey). In parallel the user named the real model: *"personas will be multiple in a conversation, like a team that forms as it goes — each one offering a different lens at each moment. Journeys and organizations tend to be more stable."* Cast vs. context. The UI rebuilt around it.

**Conversation header** replaces the rail as the home of the conversation's identity:

```
┌ Cast: (M) (T) (E) +    Context: ◈ sz  ≡ vmware-to-proxmox    Mode: auto ▾    ⋯ ┐
├───────────────────────────────────────────────────────────────────────────────────┤
│  [chat messages]                                                                   │
├───────────────────────────────────────────────────────────────────────────────────┤
│  [ type a message… ]                                                      [ Send ] │
└───────────────────────────────────────────────────────────────────────────────────┘
```

- **Cast** — mutable ensemble. Each persona renders as a circular avatar with initials + color. Click an avatar → popover with descriptor + turn count in this session + Dismiss from cast. `+` convokes a persona manually; reception keeps picking within the pool per turn.
- **Context** — stable pills. Orgs with `◈` and journeys with `≡`. Max 3 visible per group; overflow collapses to `+N more`. Click any pill → inline add/remove editor.
- **Mode pill** — click expands a four-option segmented control (auto / conversational / compositional / essayistic). Click-to-commit; no Save button.
- **⋯ menu** — New topic (POST begin-again), Look inside (admin-only, toggles the rail), Forget this conversation (with confirm).

The **rail** becomes admin-only and hidden by default. When opened, it renders a single *Look inside* panel with the composed snapshot (layers + persona + org + journey) and session stats (messages + tokens + cost + model). No more seven-block stack competing for attention.

**Message bubbles** gain a **persona signature**:
- Every assistant bubble with a persona carries a 3px lateral color bar in the persona's color. Same-persona runs keep the bar; the visual continuity tells the story without extra labels.
- A `◇ persona` text badge renders only on persona transitions (first persona in the session, or when the persona differs from the previous assistant's). Suppressed when the persona continues.

The data shape is forward-compatible with multi-persona per turn — when S5 enables reception returning `personas: string[]`, the header's Cast and the bubble signature absorb it with no rework. Voicing (integrated vs segmented) is parked for S5 too.

### Persona colors, persistent and editable

The hash-only color scheme from earlier releases becomes a stored column (`identity.color`). A native color picker on `/map/persona/<key>` lets the user choose any hex; the migration backfills existing personas with their hash-derived color so upgrades are visually invisible until the user edits.

Colors now propagate across every surface:
- **Cast avatars** (header, live-painted on reception auto-seed).
- **Bubble color bar** on every persona'd turn.
- **`◇ persona` badge** in bubbles (server-rendered + streamed).
- **Persona tag** in `/conversations` listing rows.
- **Persona card avatar** on the Psyche Map.
- **Persona listing avatar** on `/personas`.

The server is the single source of truth — the `routing` SSE event carries the resolved color so client-side paints honor the stored value (no silent divergence from server renders).

### Refinements the use surfaced

A dozen smaller moves pulled straight from the day's testing:

- **Title on first turn** — sessions earn a title right after the first user→assistant exchange, not only when Begin again closes them. Begin again still regenerates the title on the full transcript as a refinement pass.
- **Title regenerate per row** — `/conversations` listing shows a quiet `↻` on each row's top-right (on hover). Click to regenerate just that conversation's title, returning to the same filtered view.
- **Delete turn** — a small `×` appears on hover over any message bubble (user or assistant). Deletes the **turn** (user+assistant pair), not just the clicked entry, so the alternation the Agent expects stays intact.
- **Sidebar `New` entry** — between Current and See All. Same action as the header's New topic (POST begin-again), but reachable from the nav.
- **Rename labels** — *Begin again* → *New topic* (clearer intent); *Scope* → *Context* (better semantic fit for stable orgs + journeys).
- **Popovers close on click-outside** — every `<details>` in the conversation UI (menu, cast avatars, add-pickers, overflow, mode pouch) now dismisses when the user clicks elsewhere.
- **Rail POSTs respect viewed session** — tagging scopes, changing mode, or editing via the rail (now admin-only) writes to the session currently on screen, not to whatever `getOrCreateSession` returns as "current by last activity." Fixes a confusing class of cross-session writes.
- **Reception mode recalibrated** — short first-person statements about deep topics classify as conversational, not essayistic. `"Sometimes I can't understand my kids"` goes to a one-sentence reply now, not a five-paragraph essay. Form beats topic as the primary signal.

### Test growth

## Why this matters

Before v0.14.0, the mirror could already think. What it couldn't do was **speak proportionally**. A message deserving of one sentence got five paragraphs because the form rules (*how I speak*) were buried inside a thirty-layer system prompt, competing with soul and persona for attention. The pipeline turn separates the concerns: substance decides what to say, form decides how.

Before v0.14.0, the chat was a central column with a right-side rail that accumulated everything anyone ever thought to put there. The redesign honors a model the product already had but wasn't showing: **personas are voices that form a cast over time; orgs and journeys are the setting in which the conversation happens**. The header tells that story visually and in one glance.

Persona colors make the cast legible even before the user reads a word.

Pipeline patterns also set up what comes next: conditional scope activation, conditional identity layers, multi-persona turns with voicing. Every subsequent S of CV1.E7 reuses the extension point S1 and S2 installed.

## Upgrading

### From v0.13.0

```bash
git pull
npm install
systemctl restart mirror-server
```

**Schema migrations** — two new columns, both idempotent:
- `sessions.response_mode TEXT` (nullable) — CV1.E7.S1. NULL means "follow reception's mode picking." Applied via `ALTER TABLE` in `migrate()`.
- `identity.color TEXT` (nullable) — persona-colors improvement. Migration backfills existing persona rows with their hash-derived color so the upgrade is visually invisible.

**New model role** — `expression` auto-inserts from `config/models.json` via `addMissingModelRoles()` on first boot after upgrade. Defaults to Gemini 2.5 Flash with a 10s timeout. Admins who want a different model can edit via `/admin/models`.

`package.json` bumps from `0.13.0` → `0.14.0`.

After upgrade:
- No action required. Everything works on the existing data shape.
- The first time any persona'd session runs a turn, the expression pass fires. Cost bump is small (Flash is cheap) but visible in the admin usage view under the `expression` role.
- Admins who want to re-color their personas visit `/map/persona/<key>` — the picker is a native `<input type="color">` + Save. Personas that haven't been edited keep their existing hash color.
- Admins who want to inspect the composed prompt: the old rail's Composed section is still there, now under the `⋯ → Look inside` menu.
