# v0.3.0 — One Mirror, Many Tones

*16 April 2026*

**TL;DR**
- The mirror adapts tone and length to each channel — Telegram gets short conversational replies, Web gets depth
- Output formatted for each channel — Telegram MarkdownV2 with HTML and plain text fallbacks
- Per-channel instructions configurable in `config/adapters.json`
- No more wall-of-text replies on Telegram, no more raw `**markdown**` visible to users

## Where we were

v0.2.0 gave the mirror personas — it could respond as a strategist, a therapist, a technician. But it responded the same way everywhere. A question on Telegram produced the same long, structured, markdown-heavy reply as the web interface. Headers, tables, bullet lists — all of it dumped into a chat bubble. It felt like reading a document on a messaging app.

The formatting was also broken. Markdown meant for a web renderer arrived raw on Telegram. Bold markers showed as literal asterisks. The experience was functional but unpleasant.

## What we did

**We gave each channel its own voice.**

A new config file — `config/adapters.json` — defines how the mirror should communicate on each channel. Telegram gets a strict instruction: short paragraphs, conversational tone, no headers, no lists, no document structure. Web gets permission to expand. CLI gets scannable text. The instruction is appended as the last section of the system prompt, after soul, ego, and persona — a final lens that shapes how the mirror speaks right now.

This is configurable. If someone wants their Telegram mirror to be verbose, they change one line in a JSON file.

**We built a formatting pipeline.** After the LLM responds, a formatter converts the output to the channel's native format. For Telegram, this means converting markdown to MarkdownV2 (the strict format Telegram expects), with a three-tier fallback: if MarkdownV2 fails, try HTML; if HTML fails, strip all formatting and send plain text. The user always gets a readable reply — never an error, never raw syntax.

**We learned that LLMs need firm instructions.** The first attempt at "keep it short on Telegram" produced a reply with headers, tables, numbered sections, and narrated pauses ("*takes a breath before responding*"). Polite instructions don't work. The final version uses assertive constraints — "NEVER", "NO headers", "MUST". The LLM respects boundaries when they're clear.

## What comes next

The mirror now speaks differently on each channel. But the *content* is still the same identity everywhere. The next frontier is memory — the mirror remembering what matters across conversations — and journeys — the mirror knowing what you're navigating right now. When those layers arrive, the mirror won't just adapt its voice. It'll adapt its knowledge.

## Upgrading

### From any previous version

```bash
git pull && npm install && systemctl restart mirror-server
```

### Additional steps if upgrading from v0.1.0

- **Re-import your identity** to include personas: `npx tsx server/admin.ts identity import <name> --from-poc`
- **Remove `LLM_MODEL` from your `.env`** — models are now configured in `config/models.json`
