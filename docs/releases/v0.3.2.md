# v0.3.2 — One Page to See It All

*16 April 2026*

**TL;DR**
- Identity and personas unified on a single admin page per user
- Collapsible cards — name + preview when closed, full editor when open
- Old separate routes redirect to the unified profile

## Where we were

The admin had identity layers on one page and personas on another. Managing a user's full profile meant navigating back and forth. Worse, the separation hid the fact that personas *are* identity layers — just a different type. The mental model was fragmented.

## What we did

**One page, two sections.** `/admin/users/:name` now shows everything about a user's identity: base layers (soul, ego/identity, ego/behavior) at the top, personas below. Each layer is a collapsible card — closed shows the name and a content preview, open reveals a full-size editor.

Personas can be saved, deleted, or added — all inline, no page navigation. The old `/admin/identity/:name` and `/admin/personas/:name` routes redirect here.

## What comes next

The admin works. Next improvements: sidebar navigation, chat visual identity, and broader UX polish (v0.3.x series). After that, the roadmap opens to memory and journeys.

## Upgrading

### From any previous version

```bash
git pull && npm install && systemctl restart mirror-server
```

### Additional steps if upgrading from v0.1.0

- **Re-import your identity** to include personas: `npx tsx server/admin.ts identity import <name> --from-poc`
- **Remove `LLM_MODEL` from your `.env`** — models are now configured in `config/models.json`
