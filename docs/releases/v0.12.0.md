# v0.12.0 — Taking Shape

*23 April 2026*

**TL;DR**
- **[Sidebar ordering and visibility](../project/roadmap/improvements/sidebar-ordering-and-visibility/)**. Journeys and organizations gain `sort_order` and `show_in_sidebar`. Row controls on `/journeys` and `/organizations` let the user place items manually and hide them from the sidebar without archiving. Default alphabetical for existing data — seeded at migration time so the sidebar does not shuffle.
- **[Concluded status](../project/roadmap/improvements/concluded-status/)**. A third lifecycle position for journeys and organizations, distinct from archived. Concluded scopes leave the sidebar but stay routable in reception — the past still competes for attention when the message lands in its territory. Archived is reserved for the abandonment/obsolescence case.
- **Sidebar collapsible groups**. A chevron button next to Journeys, Organizations, and Psyche Map toggles the nested sub-items. State persists per group in `localStorage` so each user's layout survives reloads.
- **Psyche Map sub-links** and **read/edit mode**. The Psyche Map section expands into `Soul`, `Identity`, `Expression`, `Behavior`, and `Personas`. Clicking any of them opens the workshop in **read mode** with the content rendered as markdown; an `Edit →` link swaps in the textarea. Save redirects to the read view of the same page so the user can verify the change.
- **[`/personas` listing](../project/roadmap/improvements/personas-listing-polish/)**. Personas get their own surface paralleling journeys and organizations, each card carrying a colored circle with 1–2 letter initials from its key as a placeholder for a proper avatar.
- **[Journeys flat with org badge](../project/roadmap/improvements/journeys-flat-list-with-org-badge/)**. `/journeys` stops grouping by organization; each row carries a small pill badge naming its org (personal journeys have no badge). Reorder is flat across the full list, matching the sidebar.
- **[Regenerate Summary feedback](../project/roadmap/improvements/regenerate-summary-feedback/)** + **[Save triggers summary regen](../project/roadmap/improvements/save-triggers-summary-regen/)**. Clicking Regenerate or Save with changed briefing/situation now returns a visible banner: success, empty source, timeout, or error. The 8s title-role timeout floors at 30s for scope summaries — the role was calibrated for shorter title generation.
- **513 tests passing** (was 444 at v0.11.0). Eight distinct improvements documented under `docs/project/roadmap/improvements/`.

## Where we were

v0.11.0 (*The Memory I Carry*) made accumulated material legible — conversations by scope, across scopes, imports from prior tools. The architecture was solid, but it had never been inhabited at scale by a real user configuring a real set of scopes.

This release is that inhabitation. Every improvement is a response to friction felt while the user sat down with production and started moving scopes, naming things, archiving things, regenerating summaries, looking for prompts to read before editing them. Not roadmap-driven. Use-driven.

## What we did

### Ordering and visibility per scope

The default alphabetical order on journeys and organizations fit the empty system. The moment real scopes existed, it stopped fitting — which journey matters today is personal and alphabetical rarely matches. `/journeys` and `/organizations` rows now carry three inline controls: `↑`, `↓`, and `●/◎`. The first two reorder; the third toggles whether the item shows in the sidebar (dimmed on the listing when hidden, consistent with the archive treatment).

`sort_order` and `show_in_sidebar` land as additive columns. Existing rows receive a seeded `sort_order` matching their alphabetical position so nothing shuffles on first boot; `show_in_sidebar` defaults to 1 so every current item stays visible. The helper `moveJourney` / `moveOrganization` swap `sort_order` transactionally with the adjacent visible sibling — if either side is NULL, the swap resolves both into concrete integers so the result is stable against a third sibling.

The sidebar loader filters `show_in_sidebar = 1`. The listing page keeps every active item visible regardless, dimmed when hidden, so there is a surface to toggle the item back on.

### Concluded, distinct from archived

The existing `archived` status was doing double duty — conflating "finished its cycle" with "abandoned or obsoleted". The user noticed while trying to mark *O Reflexo* (a ship-and-done journey) without sending it to the same bucket as journeys that just fell out of rotation.

The new `concluded` status sits between active and archived:

- `active` — working on it, sidebar + reception.
- `concluded` — finished. Out of sidebar noise, still in reception (past context stays routable when the user asks about it). Visible on the listing page in its own band with a muted treatment.
- `archived` — out of routing, out of the default list, revealed via "Show archived".

No schema change (the `status` column is TEXT). New helpers `concludeJourney` / `reopenJourney` / `concludeOrganization` / `reopenOrganization`. Archive is relaxed to accept any non-archived status so `concluded → archived` is a single action. `getJourneys` / `getOrganizations` gain an independent `includeConcluded` flag; `includeArchived` narrows to mean just archived.

Reception passes `includeConcluded: true`. The sidebar default filter (`status = 'active'`) naturally keeps concluded out.

### Sidebar that takes shape

The Psyche Map section was a single link to `/map` — to reach a specific layer, the user bounced through the map dashboard every time. Five new sub-links (`Soul`, `Identity`, `Expression`, `Behavior`, `Personas`) sit under Psyche Map and navigate directly.

All three main groups (Journeys, Organizations, Psyche Map) now carry a rotating chevron. Click collapses the nested sub-items; state persists in `localStorage.sidebar-group-<name>`. The user's chosen layout survives reloads.

The Skills placeholder card on `/map` came out. It was a not-yet-built layer that didn't quite belong with self/ego/persona; when Skills takes shape later, it will live under Admin.

### Read/edit on prompt pages

The layer workshop was always an edit form — opening `/map/self/soul` meant looking at a textarea holding your own soul prompt. You couldn't *read* it without staring at an editing surface.

Now the workshop GETs a read view by default: content rendered as markdown via `marked`, an `Edit →` link in the corner. The edit mode lives at `?edit=1` and renders the existing textarea. Save redirects to the read view of the same page, which is a better loop: open to read → Edit intentionally → Save and verify.

The same page handles personas now. `isAllowedWorkshop` dynamically accepts any persona key the user has, so `/map/persona/:key` is a read/edit surface parallel to the self/ego layers. Persona save from the inline `/map` dashboard form also redirects to the new workshop, unifying the entry point.

### `/personas` surface

Personas get their own listing page at `/personas`, paralleling `/journeys` and `/organizations`. Each card carries a colored circle with initials computed from the key via the existing `avatarInitials` / `avatarColor` helpers (mentora → `ME`, product-designer → `PD`, dba → `DB`) — a placeholder for a per-persona avatar when that lands.

The first iteration shipped with reorder and sidebar-visibility controls like scopes. First use revealed a mismatch: personas don't live in the sidebar (reception picks them automatically), so the axis had no downstream effect. The controls came out; the DB columns and routes stayed (the structure is intact if a reason to bring controls back surfaces).

Individual personas no longer appear as sub-items in the sidebar — only the "Personas" sub-link under Psyche Map does. Personas are lenses, not navigation destinations.

### Journeys flat with org badge

`/journeys` used to group rows by organization. First use exposed the mismatch: most journeys are personal, single-journey groups looked overwrought, and the organization is secondary to the journey's own identity. The page is now flat. Journeys linked to an organization render a small pill badge with the org name next to the journey name; personal journeys have no badge (absence = personal).

`moveJourney` dropped its `organization_id` filter. Reorder is flat across the full per-user list, matching the sidebar and what the user sees.

### Summary regeneration that speaks

Clicking "Regenerate summary" could fail in four ways — empty briefing+situation, LLM timeout, API error, or success — and the form-POST redirect landed on the same page with the same content, making the outcome indistinguishable from "nothing happened". The server logged the failure in silence.

`generateScopeSummary` now returns `ScopeSummaryResult` — `ok` | `empty` | `timeout` | `error`. The regenerate routes echo the result as a `?summary=...` query param; the workshop renders a small banner above the Summary block with wording appropriate to the outcome. Save on briefing/situation change awaits the regen and shows the same banner (name-only edits skip the call entirely and redirect instantly). The scope-summary timeout floors at 30s, independent of the title role's 8s (the role was calibrated for title generation, not briefing+situation).

## Process note

This release codified a flexible improvement format in `docs/process/development-guide.md`. The deliberate format (plan + test-guide + refactoring) remains the standard for design-heavy work like the sidebar ordering story. A new "refinement" flavor — single `index.md` with problem, fix, commit SHA, tests — carries friction-driven fixes discovered while using the system. Seven of the eight improvements in this release ship as refinements; sidebar-ordering-and-visibility is the one deliberate entry.

The lesson: match the weight of the docs to the weight of the change. A two-paragraph index is a valid improvement entry when the fix is obvious; a three-file folder is warranted only when the reader would otherwise lose context.

## Upgrading

### From v0.11.0

git pull
npm install
systemctl restart mirror-server

Additive schema migrations applied on first boot — no manual step:

- `sort_order INTEGER`, `show_in_sidebar INTEGER NOT NULL DEFAULT 1` on `organizations`, `journeys`, `identity`.
- Seeded `sort_order` on existing rows from alphabetical position so the sidebar does not shuffle.
- `status` on journeys and organizations accepts a third value `concluded` (column stays TEXT, no ALTER).

`package.json` bumps from `0.11.0` → `0.12.0`.

Recommended (manual, after upgrade):

1. **Visit `/journeys` and `/organizations`** — use `↑ ↓ ●/◎` to set order and hide items from the sidebar without archiving.

3. **Click into a Psyche Map sub-link** (Soul, Identity, Expression, Behavior) — read view first, `Edit →` to change.
4. **Open `/personas`** — initials avatars on each card.
5. **Toggle the chevron** next to Journeys / Organizations / Psyche Map in the sidebar — state persists across reloads.
