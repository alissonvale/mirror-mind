# v0.16.0 — Every layer earns its place

*26 April 2026*

**TL;DR**

- **[CV1.E7.S3 — Conditional scope activation](../project/roadmap/cv1-depth/cv1-e7-response-intelligence/cv1-e7-s3-conditional-scope/)**: organization and journey content composes only when reception activates it for this turn. Session tags continue to constrain reception's candidate pool; they no longer force composition. A pinned scope absent from the turn's pick produces no prompt block.
- **[CV1.E7.S4 — Conditional identity layers](../project/roadmap/cv1-depth/cv1-e7-response-intelligence/cv1-e7-s4-conditional-identity/)**: `self/soul` and `ego/identity` compose only when reception flags `touches_identity: true`. New 5th axis on reception with identity-conservative defaults — silence (missing field, parse drift, failure) skips both layers; only an explicit `true` activates. Single boolean for the pair (split parked behind S4b).
- **[CV1.E7.S8 — Out-of-pool rail suggestion](../project/roadmap/cv1-depth/cv1-e7-response-intelligence/cv1-e7-s8-out-of-pool-rail-suggestion/)**: when the constrained session pool hides a strictly better candidate, reception emits `would_have_persona / would_have_organization / would_have_journey`. The bubble surfaces a non-modal *"`maker` may have something to say. [Hear it]"* card; click triggers a divergent one-turn response through the suggested persona/scope, persisted to a separate `divergent_runs` table and rendered inline as an indented sub-bubble. Session pool unchanged; the cast doesn't grow.
- **[CV1.E7.S9 — Per-turn mode visibility](../project/roadmap/cv1-depth/cv1-e7-response-intelligence/)**: `_mode` and `_mode_source` now stamped on every assistant entry meta. Bubble carries a discreet glyph for compositional (`☰`) and essayistic (`¶`) modes; conversational stays silent (presence of glyph signals escalation). The same `_mode` surfaces as a `mode: <value>` row in the Look inside composed snapshot.
- **Six refinements bundled** as S3's manual smoke surfaced latent UI / persistence drift: composed snapshot stops listing `ego.expression` (already removed from composition by CV1.E7.S1); scope pill hot-update closes the gap with `ensureCastAvatar`; auto-seed becomes per-axis with personas-asymmetric (cast can grow on any turn while pool empty; scope stays first-turn-only); bubble metadata legibility (`◈` org icon → `⌂`; mode glyph addition; conversational landing on silent after iterating through `🗨` and `“`); journey icon aligned to `↝` across all surfaces (was `≡` in the header alone).
- **CV1.E8 — Pipeline Observability & Evaluation** (new epic) opened with two drafts: S1 LLM call logging with admin toggle, S2 per-turn model switching for admin re-runs. Direct response to "I want to see what's actually happening in the pipeline" once it had grown to five named steps.
- **689 tests passing**, up from 627 at v0.15.0. +62 new across reception, identity, composed-snapshot, divergent-runs test files.

## Where we were

v0.15.0 (*A voice, a cast*) closed the gap between v0.14.0's Cast UI and the routing layer: reception returned `personas: string[]`; the composer rendered multiple lenses simultaneously under "one voice, multiple lenses"; the bubble signature handled set transitions. The cast genuinely formed and spoke as one.

What stayed unchanged: organization and journey blocks composed on every turn the session was tagged, regardless of relevance. `self/soul` and `ego/identity` composed on every turn, period. The pipeline had become smarter about *who* was talking, but not yet about *what context* the turn actually warranted. Pool-as-constraint kept conversations focused but silently locked out genuinely better picks when a turn drifted outside the declared frame — and the user had no signal that the lockout existed.

v0.16.0 is the response on three fronts: stop carrying scope content when the turn doesn't ask for it (S3), stop carrying identity content when the turn doesn't ask for it (S4), and admit the pool's lockout instead of producing a misframed canonical answer (S8). The fourth front is making the per-turn decisions visible (S9) so the user can verify the pipeline did what they expected.

## What we did

### Conditional scope (S3)

The composer used to treat session tags as a force-include: any tag, on every turn, in the prompt. S3 inverts the priority: reception's pick is the source of truth; tags constrain reception's candidate pool only. A small-talk turn on a session pinned to `software-zen` and `vida-economica` no longer carries either briefing+situation block — the prompt opens with the persona (if any) and the form cluster, nothing more.

The conversation header pill (session-level, stable) and the bubble badge (per-turn, conditional) now agree with what the prompt actually carried. The "Look inside" snapshot matches turn-by-turn reality.

[5 pre-S3 tests deleted (defunct "tag forces always" semantics); 4 new tests added pinning reception-as-source-of-truth.]

### Conditional identity (S4)

`self/soul` (essence, frequency, principles) and `ego/identity` (operational positioning, stances) are the heaviest two layers in the prompt — together a few thousand tokens of identity content. Until S4, they composed on every turn whether the turn invited that depth or not.

S4 adds a fifth axis to reception: `touches_identity: boolean`. The pair gates together — the layers are conceptually adjacent (essence + operational positioning are "who I am" on a spectrum); splitting parked behind S4b. Identity-conservative defaults — silence in any form (missing field, parse drift, reception failure) skips both layers. Only an explicit `true` activates.

A casual greeting now composes just `ego/behavior` plus the adapter instruction. A reflective question about identity / purpose / values composes the full identity cluster. The token cost shifts proportionally; the tonal cost is the bigger gain — operational turns no longer respond with existential framing.

[8 new tests in reception, 8 in composer, 4 in snapshot.]

### Out-of-pool rail suggestion (S8)

S3's manual smoke surfaced the canonical use case empirically: Dan asked about Stanley plane comparison on a session whose cast was constrained to `[engineer]`. Reception had no choice but to activate `engineer` (the only candidate). The response came back competent-but-misframed — engineer's lens applied to woodwork. Dan had no signal that `maker` existed in his data and would have been the right voice.

S8 makes the lockout visible and opt-in. Reception now emits three new "would have picked" axes — `would_have_persona`, `would_have_organization`, `would_have_journey` — populated only when the out-of-pool candidate is a strictly better fit than the in-pool options. The bubble surfaces a non-modal card below the canonical: *"`maker` may have something to say. [Hear it]"*. Click triggers a divergent run.

The divergent path:
- New `POST /conversation/divergent-run` endpoint composes a system prompt with the override, runs main + expression, persists to a new `divergent_runs` table.
- The response renders as a sub-bubble inside the parent's `.msg-body`, indented with the override's badge and (for personas) color bar — visually clear that it's a side branch.
- Session pool stays unchanged. The cast doesn't grow on click; reception doesn't start surfacing the same suggestion everywhere because it once worked. The next turn operates against the same pool as before.
- `divergent_runs` lives in a separate table from `entries`, so the agent's history feed (`loadMessages`) automatically excludes divergent runs without any filter logic. The next turn's LLM doesn't see them.

The smoke test fixture from S3 reproduced exactly: engineer answered the Stanley plane question with "different lengths"; the suggestion card fired; clicking it produced maker's response — *"smoothing plane vs jack plane (sometimes called fore plane), aggressive stock removal vs finish work"*. Two responses on the same turn, two lenses, one cast unchanged.

[8 new tests in reception (would_have_X axes + drift guards + prompt structure); 11 in divergent-runs (insert/load/cascade/scoping/history exclusion).]

### Per-turn mode visibility (S9)

`_mode` and `_mode_source` now stamped on every assistant entry's meta. The bubble carries a discreet glyph at the start of its text — `☰` for compositional, `¶` for essayistic. Conversational stays silent on the bubble (it's the default; presence of any glyph signals escalation). The same `_mode` surfaces as a `mode: <value>` row in the Look inside composed snapshot, so the rail's diagnostic view now matches the bubble's visual signal turn-by-turn.

The conversational glyph went through three iterations before landing on silence: started as `🗨` (filled emoji speech bubble — broke the line-art family), swapped to `“` (quotation mark — read as "incoming text from elsewhere", didn't fit the close register), finally settled on omission (visual silence carries the meaning). The metadata family at the bubble's edge now reads as one coherent line-art set: `◇` persona, `⌂` org, `↝` journey, `☰` compositional, `¶` essayistic.

[Three new tests in composed-snapshot for the mode field; existing tests updated.]

### Refinements bundle

S3's manual smoke surfaced six latent UI / persistence drifts the prior tightening had hidden. Each landed as a focused refinement:

- **Composed snapshot stops listing `ego.expression`.** CV1.E7.S1 moved expression to the post-generation pass; the snapshot's layers list was still reading from the DB inventory. Filter added; honest snapshot.
- **Scope pill hot-update.** `ensureCastAvatar` was symmetric counterpart to nothing — scope pills fell to the server-render of next page load. Added `ensureScopePill` triggered by an explicit `seededScopes` field on the routing event. Now scope pills appear in real time on first-turn auto-seed, just like Cast avatars.
- **Auto-seed per-axis.** The all-or-nothing `hasAnyTag` gate suppressed persona seeding when org/journey were pinned. Pass 1 split per-axis (each axis seeds independently on first turn while its own pool is empty). Pass 2 made the persona axis asymmetric: cast can grow on any turn while its pool is empty, matching the cast-vs-scope mutability principle from CV1.E7.S2.
- **Bubble metadata legibility.** Org icon `◈` (filled diamond, visually too close to `◇` persona) swapped to `⌂` (line-art house). Mode glyph added at start of bubble. Three iterations on the conversational glyph until landing on silence.
- **Journey icon aligned.** Was `↝` everywhere except the conversation header (`≡`). Aligned to `↝` across all surfaces.
- **S3 test guide expanded.** Dan Reilly's section grew from a four-row message table into a 20-step worked walkthrough; Elena/Eli/Nora kept the compact form, inheriting the procedure from Dan's example.

### CV1.E8 opened

Once the pipeline had five named steps and out-of-pool detours, the natural next question became: *what's actually happening inside?* CV1.E8 — Pipeline Observability & Evaluation — opened as a new epic with two drafts:
- **S1 — LLM call logging with admin toggle.** Every model invocation writes to a log table (model, prompt, response, tokens, cost, latency). Admin toggle to start/stop. Foundation for finding optimization points.
- **S2 — Per-turn model switching for admin re-runs.** Admin re-runs any past turn through a different model; alternative response renders alongside for comparison. Doesn't mutate the canonical conversation.

Both drafts. Implementation deferred until the dor surfaces.

## What this means in practice

A session pinned to a project's org and journey, used for a casual Friday-evening check-in (*"long week, going to bed"*), now responds with just `ego/behavior` plus the adapter instruction. The full identity cluster (soul + identity) and the project's briefing+situation no longer load. The reply is short and close, matching the form the user used.

The same session, the next morning, asked about the project's state, composes the full identity cluster + the org's briefing + the journey's situation + the right persona. The mirror's depth shows up where the depth is asked for.

The same session, a week later, asks something outside the project's domain — *"can you help me with this Haskell question?"*. Engineer can't cover it cleanly; reception flags `would_have_persona: tecnica` (or whichever covers programming in the user's library); the bubble offers *"`tecnica` may have something to say"*; click produces a tecnica-flavored answer inline. The cast stays as it was; the session continues.

The pipeline's volume — and the prompt's tokens — track what the turn actually asked for.
