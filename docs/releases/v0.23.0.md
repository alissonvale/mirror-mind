# v0.23.0 — The mirror has a face

*4 May 2026*

**TL;DR**

- **CV1.E12 epic closed** — three stories ship the contemplative entry-point at `/`:
  - **S1 — Chrome inversion + page skeleton.** The `◆ Mirror Mind` logo flips from "start a conversation" to point at `/espelho` (the page that finally honors the brand name). New `▶ Iniciar` pill takes over the operational role the logo used to carry. Avatar dropdown gains an `Início` text shortcut. No `Espelho` item in the menu — the logo IS the entry, on purpose.
  - **S2 — Living synthesis (glance + shifts + Sou/Estou/Vivo).** New `server/mirror/synthesis.ts` composes a typed `MirrorState` from current data: active voices over the last 7 days, focus journey of the week, dominant org, recurring themes (≥2 sessions), week conversation count + day count, last titled session. Plus a "what shifted since last visit" pulse that diffs against new `users.last_mirror_visit_at` column (soul-updated, new-journey, scene-reopened, many-conversations markers). Three depth cards render the synthesis: `Vivo` / `Estou` / `Sou` (column order inverted from the original draft after design conversation — temporal flow from "what's moving" to "what holds still"). Each card carries a question subtitle ("o que estou vivendo?", "por onde opero?", "quem sou?"), an axis-specific glyph (◌ ◉ ✦), an axis color accent (plum / teal / amber border-left), a heading link to the drill-down surface (`/memorias`, `/territorio`, `/map`), and prose or numeric tiles. Tile numbers + labels link to listing pages (`/journeys`, `/cenas`, `/conversations`). Every entity name in the prose is wrapped in an `<a>` with a subtle dotted underline — focus journey → `/journeys/<key>`, persona name → `/map/persona/<key>`, recurring theme → typed URL, last conversation → `/conversation/<id>`, etc. Active voices line replaces the original single dominant-voice tag: lists every voice the user spoke through this week (Alma + each persona tagged), capped at 4, with `♔` (alma) or `◇` (persona) glyphs and persona names title-cased. Shifts also link (soul-updated → `/map/self/soul`, new-journey → `/journeys/<key>`, etc).
  - **S3 — Ímãs (pinned phrases).** Originally named "Inscrições"; renamed to "Ímãs" / "Magnets" mid-implementation because the fridge-magnet metaphor translates 1:1 across PT/EN and reads warmer than "inscriptions". New `inscriptions` table (the data-layer name stays — developer concept of "what the user has inscribed") + `pickInscriptionForToday` resolver: most recent pinned wins, otherwise deterministic daily rotation (djb2 hash of `userId + UTC ymd` mod active count). Render block at top of `/` with a soft cream tile, faded amber border, a tiny amber dot on top center as the metaphorical magnet, italic serif body in EB Garamond with curly quotes, em-dash + author below in sans. Management page at `/espelho/imas` (kept on the legacy /espelho path even after the URL swap): list of active ímãs with per-row pin/unpin/edit/archive actions; archived band collapsed via `<details>`; quiet inline edit forms also via `<details>`; no JS — every action is a tiny POST + redirect. Avatar dropdown gains an `Ímãs` entry between Conversations and Skills (later removed in the menu reorder). PT verbs adapted to the magnet metaphor: "fixar / soltar / guardar" instead of "fixar / despinar / arquivar". A second "rotating magnet" surfaces at the bottom of Vivo under a `RELEMBRE` label — daily rotation from the non-pinned pool, excluding whatever id the top tile resolved to so the two surfaces never double-show. New `pickRotatingMagnetForToday(db, userId, now, excludeId)` picker.

- **URL swap: `/` is the Espelho.** Operational home moves from `/` to `/inicio`; the mirror page that lived at `/espelho` becomes `/`. Logo brand link updates accordingly; `▶ Iniciar` pill points at `/inicio`; avatar dropdown's `Início` item points at `/inicio`. `/espelho` 301-redirects to `/` for any bookmark stragglers. Tests updated across inicio-routes, espelho-routes, imas-routes, web (the `/inicio (legacy)` redirect test now covers `/espelho → /`), cenas-routes (delete-redirect now goes to `/inicio`).

- **CV1.E11 follow-up: Território splits out of Memórias.** The `/memorias` dashboard from CV1.E11.S3 was conflating two timeframes — the lived record (history + future Library) and the present-active world (cenas, journeys, organizações). New `/territorio` route owns the entity cards; `/memorias` slims to Library + Histórico. Avatar menu adds Território between Cognitivo and Memórias (later reordered to Início → Território → Mapa Interior → Memórias). New i18n namespace `territorio.*`; `memoria.*` trimmed. Tests split: `tests/territorio-routes.test.ts` is new; `tests/memoria-routes.test.ts` rewritten for the slim page.

- **Naming polish.**
  - `Mapa Cognitivo` → `Mapa Interior` ("Cognitive Map" → "Inner Map"). "Cognitivo" leaned too academic; "Interior" reads quieter and dialogues with the `/espelho` Sou pane's "quem sou?" subtitle. Renamed across `topbar.menu.cognitive`, `map.htmlTitle`, `map.identityPrefix`, `layer.breadcrumbBack`. URL `/map` stays — abstract enough to outlive the naming shift.
  - `Inscrições` → `Ímãs` ("Inscriptions" → "Magnets") as documented in the S3 entry above.

- **Data-layer fixes that unblock cena-first usage.**
  - **Cena `response_mode` + `response_length` propagate into sessions on start.** The cena form already lets you pick conversational/essayistic/etc as defaults; the fields were saved on the scene row but never read on `/cenas/:key/start`. Same handler already propagated cast (personas), voice (alma), organization_key, and journey_key. Adding the response-axis seeding finishes the contract: everything the cena promises is active from turn 0.
  - **Org / journey / layer summaries are now editable on the workshops.** Previously the summary was always auto-regenerated by the model on Save. Now the workshop form carries the summary as a regular textarea (with a Regenerate button next to its label using `formaction=` to POST to the regenerate endpoint — no nested forms). User text wins; blank + briefing/situation change → regenerate; blank + no change → leave whatever's there. Applied to organizations, journeys, and layer-workshop (self/ego/personas).
  - **`/espelho` conversation count matches `/conversations`.** Vivo's `weekConversationCount` and `weekDayCount` were counting every session created in the last 7 days, including ghost sessions (createFreshSession with no follow-up entry — failed LLM call, mid-stream abandons). The `/conversations` listing, `/memorias` Histórico, and Recentes all filter ghosts via `EXISTS (SELECT 1 FROM entries WHERE session_id = s.id)`. The mismatch surfaced as "the espelho says 8 conversations, but the list page shows 5." Same EXISTS filter applied to sessionsThisWeek query, many-conversations shift, and scene-reopened detection.

- **Chrome unification fix uncovered a silent system-wide CSS bug.** The legacy `Layout` component (used by `/me`, `/map`, `/personas`, `/organizations`, `/journeys`, `/conversation`) carried its own *parallel, stale* copy of the avatar-top-bar CSS rules. When CV1.E12.S1 added the new `Iniciar` pill class to AvatarTopBar, only `TopBarLayout`'s inline copy got the corresponding styles — `Layout`'s didn't. Pill rendered as a default underlined `<a>` on every legacy page. Fix: extract `AVATAR_TOP_BAR_STYLES` constant exported from `avatar-top-bar.tsx`; both `Layout` and `TopBarLayout` pull from it. Single source of truth; future chrome additions land in one place. **Bigger discovery:** while debugging why EB Garamond wasn't applying to the soul prose, a DevTools inspection revealed the rendered font was `.SFNS-RegularItalic` (system sans) on a paragraph that declared `font-family: 'EB Garamond', 'Baskerville', serif`. Root cause: hono/jsx HTML-escapes string children of `<style>` tags by default — `'` becomes `&#39;`, every quoted font-family declaration parsed as invalid CSS and the browser silently dropped them. The italic styling kept landing (font-style is a separate declaration), which masked the failure for months — text looked italic, just wrong-italic. **Sweep applied across every inline `<style>{...}</style>` in the codebase** (espelho, espelho-imas, layout, avatar-top-bar, territorio, memoria, home-inicio, cenas-list, cenas-form): wrap content in `raw()` from `hono/html`. Now every quoted font name reaches the browser verbatim.

- **Sou pane: noble typography for the soul summary.** Voice line cut from Sou (already in glance). Body shifted to **EB Garamond italic** loaded as a Google web font, with a 2.6em warm-amber **drop-cap** on the first character (rendered roman for the classic chapter-opening contrast). Stack falls back through Baskerville → Hoefler Text → Cambria → serif while the woff2 downloads. Calibration journey took several rounds: Iowan Old Style → Hoefler Text → Baskerville → EB Garamond, with a temporary `/espelho/font-preview` route built to compare options side-by-side (including a JS canvas font-detection probe that revealed the real cause — hono escape — once the user reported all options looked the same).

- **Visual system on `/espelho`.** Three columns get axis colors (amber Sou, teal Estou, plum Vivo), per-axis glyphs (`✦ ◉ ◌`), and a soft cream card background with colored left accent. Numeric tiles (large display number 300-weight + small caps label + italic sub-line) replace count-as-prose for travessias / cenas vivas / conversas. Pane subtitles answer the user's three foundational questions in italic serif. Glance line was originally at the top in serif italic but later removed entirely — voice + focus journey moved into Vivo (where they belong as activity signals), soul summary already lived in Sou, so the glance was redundant. Column order inverted to Vivo / Estou / Sou in the same refactor.

- **Avatar dropdown reorder.** Final order: `Início → Território → Mapa Interior → Memórias` (drops Conversas and Ímãs). Both still reachable: Conversas via the conversation count tile in Vivo and "ver tudo →" links in Memórias/Recentes; Ímãs via the discrete footer link on `/`.

- **Tests: 1097 passing**, up from 1022 at v0.22.0. New unit + route suites: `tests/territorio-routes.test.ts`, `tests/espelho-routes.test.ts`, `tests/espelho-synthesis.test.ts`, `tests/inscriptions.test.ts` (CRUD + picker), `tests/imas-routes.test.ts` (management routes). Existing suites updated for the URL swap, the editable summary fields, the cena response_mode/length propagation, and the active-voices list replacing the dominant-voice tag.

## What we did

### CV1.E12 — The Mirror page (the contemplative entry-point epic)

Three stories trace the full implementation from chrome inversion to living synthesis to inscriptions.

**S1 — Chrome inversion + `/espelho` skeleton.** The `◆ Mirror Mind` logo was conflating brand identity with the operational verb "iniciar" — clicking it landed on the cena-cards home, not on anything the brand name suggested. Adding a contemplative surface on top of that conflict was guaranteed to feel like duplication. The fix:

- Logo `href`: `/` → `/espelho` (later `/` after the URL swap)
- New `<a class="avatar-top-bar-start" href="/">▶ Iniciar</a>` pill in the chrome, immediately right of the brand
- Avatar dropdown gains an `Início` item pointing at the operational home
- `EspelhoPage` placeholder shell at the new route — TopBarLayout + minimal body until S2

The pill takes over the operational role; the brand becomes the doorway to what the brand says.

**S2 — Living synthesis.** The page reads top-to-bottom as one self-portrait covering three questions: where I'm living, where I operate, who I am. Implementation across three layers:

- **`server/mirror/synthesis.ts`** — pure compose functions returning typed `MirrorState`:
  - `composeSou(db, userId)` — soul summary (with `pickSummaryOrFirstSentence` fallback)
  - `composeEstou(db, userId, now)` — active journeys, dominant org of the week, active scene count, most recent scene
  - `composeVivo(db, userId, now)` — active voices (alma + each tagged persona, capped at 4), focus journey of the week, recurring themes (≥2 sessions over 7 days, scene/org/journey), week conversation count, distinct day count, last titled session
  - `computeShifts(db, userId, lastVisit, now)` — diff against `users.last_mirror_visit_at`: soul-updated (with daysAgo), new-journey, scene-reopened, many-conversations
  - `composeMirrorState` — orchestrator
- **DB:** new `users.last_mirror_visit_at` (idempotent ALTER); helpers `getLastMirrorVisit` / `setLastMirrorVisit`
- **Page:** three depth cards in a CSS grid (auto-fit, minmax 260px), each with axis color (amber/teal/plum border-left), per-axis glyph (`✦ ◉ ◌`), heading link to drill-down, italic serif question subtitle, body of prose + numeric tiles + tag lines

The visual system iterated several times during the session in response to user feedback:

- First cut had three columns of paragraphs — felt like a wall of text. Refined with axis colors, glyphs, numeric tiles for counts (large display number 300-weight + small caps label + italic sub).
- Original Sou body rendered both soulSummary and identitySummary stacked. Trimmed to soulSummary only; the deep-identity statement is the anchor.
- Original column order was Sou / Estou / Vivo; user requested inversion to Vivo / Estou / Sou after observing the temporal flow ("what's moving" → "what holds still").
- Glance line at the top was removed entirely — voice + focus journey moved into Vivo (activity signals belong there), soul summary already in Sou, so glance was redundant.
- Voice originally rendered as single "dominant voice" tag (one of alma or persona). Refined to **active voices list**: every voice the user spoke through this week, with `♔` for alma and `◇` for personas, comma-joined with a final "e"/"and" via new `common.lastSeparator` i18n key. Persona keys title-cased on display (`product-designer` → `Product Designer`).
- Every entity name in the synthesis became a link with a subtle dotted underline. Helper `withLink(templateKey, varName, varValue, href, extraVars)` renders i18n templates with one variable replaced by an inline `<a>` using a Unicode private-use sentinel split. Tile numbers + labels link to the corresponding listing page.

**S2 noble typography for Sou.** The soul summary deserved more contemplative typography than body serif. After several rounds (Iowan Old Style → Hoefler Text → Baskerville), the user requested EB Garamond loaded as a Google web font for cohesion across devices. The first deploy didn't show any change — DevTools revealed the rendered font was `.SFNS-RegularItalic`. Cause: hono/jsx HTML-escape on `<style>` children was killing every quoted font-family declaration. Fixed by wrapping all inline styles in `raw()`. The italic landed; the warm-amber 2.6em drop-cap rendered roman for the classic illuminated-letter contrast.

**S3 — Ímãs (pinned phrases).** Originally named "Inscrições", renamed to "Ímãs" / "Magnets" mid-implementation. The fridge-magnet metaphor translates 1:1 across PT/EN and reads warmer than "inscription" without losing any flavor. Schema: new `inscriptions` table (id, user_id, text, author, pinned_at, created_at, archived_at, all timestamps INTEGER milliseconds). Two pickers:

- `pickInscriptionForToday` — most recent pinned wins; otherwise deterministic daily rotation (djb2 hash of `userId + UTC ymd` mod active count); null if no active inscriptions.
- `pickRotatingMagnetForToday(db, userId, now, excludeId)` — added later for the Vivo "RELEMBRE" surface; only considers non-pinned, optionally excludes one id (typically the top inscription's id) so the two surfaces never double-show on a day with no pinned magnet.

Render at top of `/`: soft cream tile (`#fdf8ec`) with faded amber border, tiny amber dot top center as the magnet glyph, italic serif body with curly quotes via `quotes` CSS, em-dash + author in sans. Empty `<aside>` collapses via `:empty` when no ímã surfaces. Management page at `/espelho/imas` (path kept on the espelho prefix even after the URL swap): list + add form + per-row pin/unpin/edit/archive actions; archived band collapsed via `<details>`; inline edit also via `<details>`; no JS — every action is a tiny POST + redirect; ownership-checked at every mutation (foreign id → 404). Discrete footer link on `/` (`#a0aec0` text). Action verbs in PT shifted to the magnet metaphor: "fixar / soltar / guardar" instead of "fixar / despinar / arquivar".

A **rotating magnet surface** added at the bottom of Vivo under a "RELEMBRE" label (originally placed at the top, then moved per user request). Same caps-tracked typography as the tile labels. Magnet text in EB Garamond italic. The user's collection rotates daily through this surface even when one ímã is pinned at the top — the top is "the anchor I chose to fix", Vivo's RELEMBRE is "the daily whisper from your collection".

### CV1.E11 follow-up — Território splits out of Memórias

`/memorias` was conflating present-active context (cenas, journeys, orgs — the world the user moves in *now*) with past record (history + future Library). The split:

- New route `/territorio` owns the three entity cards (TerritorioPage component, `territorio.*` i18n namespace).
- `/memorias` slims to Library + Histórico; the entity dashboard moves out.
- Avatar menu inserts Território between Cognitivo and Memórias.
- Tests split: new `tests/territorio-routes.test.ts`; `tests/memoria-routes.test.ts` rewritten for the slim page.

### URL swap: `/` becomes the Espelho

After CV1.E12 shipped, the brand-vs-URL conflict was still present at the URL level: the logo pointed at `/espelho` while `/` served the operational home. Swap formalized:

- `/` → EspelhoPage (the mirror, default landing)
- `/inicio` → InicioPage (operational home, cards + free input + recents)
- `/espelho` → 301 redirect to `/`

Chrome updated: brand `href` flips to `/`; pill flips to `/inicio`; menu Início flips to `/inicio`. Form actions, redirects, tests updated. Login redirect stays on `/` — landing on the mirror after auth is coherent with the door-then-mirror metaphor; the user can click Iniciar immediately if they want to work.

### Naming refactors

- **Mapa Cognitivo → Mapa Interior** ("Cognitive Map" → "Inner Map"). Across `topbar.menu.cognitive`, `map.htmlTitle`, `map.identityPrefix`, `layer.breadcrumbBack`. URL `/map` stays.
- **Inscrições → Ímãs** ("Inscriptions" → "Magnets"). Across all `espelho.imas.*` keys, the menu, the page (`espelho-inscricoes.tsx` → `espelho-imas.tsx`; `InscricoesPage` → `ImasPage`; routes `/espelho/inscricoes/*` → `/espelho/imas/*`). PT verbs adapted to the metaphor. The data layer keeps `inscriptions` as the table name (developer concept).

### Data-layer fixes

- **Cena response_mode + length propagate to session on start.** Two new tests in inicio-routes cover both the propagation and the null-stays-null case.
- **Editable summary on workshops.** Org, journey, and layer (self/ego/personas) workshops now carry the summary as a regular textarea with a Regenerate button using `formaction=` (no nested forms). User text wins; blank + content change → regenerate; blank + no change → leave. New i18n key `scope.workshop.summaryHint` and `layer.summaryEditHint`. New CSS class `.workshop-summary-head`.
- **Conversation count consistency.** `composeVivo`'s `sessionsThisWeek` query, `computeShifts`'s `many-conversations` count, and the `scene-reopened` detection all gain the same `EXISTS (SELECT 1 FROM entries WHERE session_id = s.id)` filter that `/conversations` and `/memorias` use. Now the same set of sessions is counted everywhere.

### Chrome unification + raw() sweep

- **Avatar-top-bar CSS unified.** Extract `AVATAR_TOP_BAR_STYLES` exported from `avatar-top-bar.tsx`; both `Layout` and `TopBarLayout` consume it. Layout keeps a small additional `<style>` block for legacy-only concerns (wide-mode flag, inner-content centering). The Iniciar pill now renders correctly on every page.
- **`raw()` applied to every inline `<style>{...}</style>`** in `avatar-top-bar.tsx`, `layout.tsx`, `espelho.tsx`, `espelho-imas.tsx`, `territorio.tsx`, `memoria.tsx`, `home-inicio.tsx`, `cenas-list.tsx`, `cenas-form.tsx`. The hono/jsx escape was killing every quoted font-family in CSS rules — silently. Fonts that should have been loading for months (Iowan Old Style, Charter, Georgia, etc.) were falling back to the body's sans-serif. Fix is one line per file (import raw from hono/html, wrap the inline content). Now every typography choice in the system actually applies.

### Avatar dropdown reorder

Final shape: `Início → Território → Mapa Interior → Memórias`. Drops Conversas and Ímãs. Conversas reachable via the count tile in Vivo and "ver tudo →" links elsewhere; Ímãs via the discrete footer link on `/`. The four-item menu maps cleanly to the three axes plus the operational entry — one entry for each kind of thing the user might want to navigate to from the chrome.
