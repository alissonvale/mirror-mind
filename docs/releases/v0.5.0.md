# v0.5.0 — The Mirror Shows Itself

*18 April 2026*

**TL;DR**
- `/map` — the Cognitive Map: self, ego, personas, skills as a workspace with per-layer workshops, live composed-prompt preview, and a lateral memory column
- `/mirror` — the chat page gains the Context Rail (attention memory visible) and a proper name (replacing `/chat`, which stays as a redirect)
- Identity + authority in the sidebar — avatar, name, role-aware menu, 403 on `/admin/*` for non-admins
- Self-service identity editing — users write their own soul, identity, behavior, personas, and display name
- Empty-state invitations — each empty layer teaches what it is and invites the user to shape it
- Identity layers now ordered by psychic depth (self → ego → persona), not alphabetically
- 55 new tests (123 total), CV0.E2 epic closed

## Where we were

v0.4.0 gave the web its own home — pages moved into `adapters/web/`, a sidebar replaced the top nav, the chat got warm colors and rendered markdown. But the mirror itself stayed opaque: its identity lived in DB tables invisible to the user, its attention was a server-side composition no one could see, and every edit happened through an admin screen that treated the user's own soul like someone else's homework.

The user couldn't see the mirror. The mirror couldn't show itself.

## What we did

**We opened `/map`.** The Cognitive Map is a workspace where the psyche's structure becomes visible and editable. Self at the top (essence), Ego as a side-by-side pair (identity + behavior), Personas as a single card with badges (one persona per badge, inline editor when you click), Skills at the bottom (action — still empty, invitation shown). Clicking a structural card opens its **Layer Workshop** — a focused page with a generous editor on the left and a live composed-prompt preview on the right that updates as you type, no LLM call, no cost. The user sees the effect of each word on the system prompt the mirror reads. Every card, when empty, carries a paragraph that answers *what is this layer* and *what do I do with it*.

**We added the Context Rail.** The chat page (now `/mirror`) gained a right-side panel showing the mirror's **attention memory** — which persona answered, session stats (tokens, cost, model), and the identity layers composed into this turn's prompt. A window into what the mirror is holding right now. Collapsible, persisted per user.

**We made memory a column, not a row.** On the map, a narrow column on the right lists memory's three faces — attention (→ rail), conversations (episodic, coming), insights (semantic, coming). The column sits *perpendicular* to the structural stack because memory traverses every psychic layer rather than following them: self has memory, ego has memory, personas have memory. Placing it alongside the whole stack honors that geometry. The rail and the memory column both live on the right periphery — "what the mirror carries" has a consistent home across surfaces.

**We made identity and authority visible.** The sidebar shows your avatar and name; admin sees a Users link, regular users don't. Non-admins hitting `/admin/*` get `403 Forbidden` — an honest boundary, not a friendly redirect. You can edit your own display name inline from the map. Under the hood, a `role` column joined the users table, and existing installations automatically promote the oldest user to admin on first boot — zero friction upgrade, no SQL required.

**We gave self-service its full scope.** A user can now edit their own self, ego, personas, and name without needing an admin. "My map is mine" — the Workshop's live preview makes consequences visible in real time, which mitigates the old concern that untutored users might degrade their own mirror. Admins still create and delete users; they can also view and edit other users' maps at `/map/<name>`, with a "viewing as admin" banner and the name-edit affordance suppressed.

**We retired the admin profile editor.** `/admin/users/:name`, `/admin/identity/:name`, and `/admin/personas/:name` all redirect to `/map/:name`. The old UserProfilePage is deleted. Bookmarks still work; the experience is unified.

**We renamed `/chat` to `/mirror`.** The page is where you meet the mirror, not a chat window. The chat affordance is one element inside it, alongside the rail and the composed identity. `/chat` stays as a redirect so shared links never break.

**We fixed the composition order.** Identity layers used to be returned alphabetically — `ego/behavior` led the composed system prompt, `self/soul` ended it. Surfacing the prompt in the Workshop preview exposed the inversion. The SQL now orders by psychic depth: `self` (essence) → `ego` (operational) → `persona` (specialized voice). Every consumer inherits the correct narrative order; the composed prompt now opens with the foundation, ends with the behavior that emerges from it.

**We changed what happens when an admin creates a user.** Only `ego/behavior` is seeded from a template — the operational baseline the mirror needs on turn one. `self/soul` and `ego/identity` are left empty on purpose, so the Cognitive Map's invitations appear on the new user's first visit and teach them what those layers are. The user writes their own voice, not a template's.

**We adopted push-at-story-completion.** Completed stories now push to `origin` as their last act. Release is a tag event, not a push event. Doc-only and backlog commits ride along with the next story's push. Escape valve: push a `wip/<story>` branch when a story spans sessions and needs remote backup mid-flight.

## What comes next

Two directions are registered and waiting, either can come next:

- **CV0.E3 — Install Administration.** The admin operates the install from the browser, not from JSON files. Three stories: customize models (S1), customize adapters (S2), and read the mirror's own documentation inside the app (S3). The first two move `config/models.json` and `config/adapters.json` into a live DB-backed admin surface so tuning happens without restarts. S3 puts `docs/` at `/docs` with a navigable sidebar and live rendering — the project explains itself from inside.

- **CV1.E3.S4 — I can reset my conversation.** Two modes: start a fresh session (previous preserved for future recall) and, as a destructive mode, delete the current session's entries entirely. Registered ahead of CV1.E3.S1 (automatic topic-shift detection) because manual boundary-setting teaches the system and the user what a session boundary means before the automated version calibrates.

A third direction — **Identity Workshop test chat** — sits in the CV0.E2 radar: the Workshop page gains a mini-chat where the user sends messages and sees the mirror respond using their *draft* identity before saving. Substantial enough to deserve its own design conversation.

## Upgrading

### From any previous version

```bash
git pull && npm install && systemctl restart mirror-server
```

On startup, the migration in `openDb` runs automatically:
- Adds a `role` column to `users` if absent (default `'user'`).
- Promotes the oldest user by `created_at` to `admin` when no admin exists yet, so existing installations keep functional admin access.

No data changes to existing identity layers. Users who previously had `self/soul` and `ego/identity` populated from templates still have that content — the seed-only-ego/behavior rule applies only to *new* user creations.

### Additional steps if upgrading from v0.1.0

- Re-import identity: `npx tsx server/admin.ts identity import <name> --from-poc`
- Remove `LLM_MODEL` from your `.env` (models are configured in `config/models.json`)

### Notes for admins managing other users

The admin profile page is gone. Use `/map/<name>` to view or edit another user's identity layers and personas. Name edits are still self-only — admins can't rename other users from the map. (Admin renaming is a future concern.)
