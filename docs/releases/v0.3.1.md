# v0.3.1 — Polish and Clarity

*16 April 2026*

**TL;DR**
- Admin personas page — view, edit, add, and delete personas per user from the browser
- Prompt composition reference — documentation showing exactly how prompts are built for each scenario, with full examples
- Release notes navigation — prev/next links for sequential reading

## Where we were

v0.3.0 shipped adapter awareness — the mirror adapts its tone per channel and formats output correctly. But managing personas required the admin CLI. And the inner workings of prompt composition were undocumented — a contributor couldn't answer "what exactly does the LLM see?" without reading code.

## What we did

**We made personas manageable from the browser.** A dedicated admin page per user lists all their personas, lets you edit content inline, add new ones, and delete. No CLI needed for day-to-day persona management.

**We documented the prompt pipeline.** A new reference in `docs/product/prompt-composition/` explains the composition order (soul → ego → persona → adapter instruction), shows where each piece is configured, and includes three full example prompts — the actual text the LLM receives, built from starter templates. No personal content, fully generic.

**We connected the release notes.** Each release now links to the previous and next, so you can read the story sequentially without returning to the index.

## What comes next

The foundation is solid. The next big step is **memory** — the mirror remembering what matters across conversations. After that, **journeys** — the mirror knowing what you're navigating right now.

## Upgrading

### From any previous version

```bash
git pull && npm install && systemctl restart mirror-server
```

### Additional steps if upgrading from v0.1.0

- **Re-import your identity** to include personas: `npx tsx server/admin.ts identity import <name> --from-poc`
- **Remove `LLM_MODEL` from your `.env`** — models are now configured in `config/models.json`
